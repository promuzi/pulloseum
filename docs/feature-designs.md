# 풀로세움 — 8대 기능 상세 설계서 (구현 직전 단계)

> **이 문서는 "기획만" 단계입니다.** 게임 코드는 아직 손대지 않았습니다.
> [benchmark-proposals.md](benchmark-proposals.md)의 ①~⑧ 항목을 **하나하나 바로 구현 가능한 수준**으로 풀어 적었습니다.
> 각 항목마다 **선택지별 동작 / UI / 데이터 모델 / 구현 방식(실제 함수·state 필드 기준) / 난이도·리스크**를 넣었습니다.
> **고르는 법:** `①A 채택`, `③A로 가되 보상은 미네랄 말고 종자로` 처럼 **번호+알파벳 + 수정사항**을 말하면 됩니다. 고른 것만 [master-roadmap.md](master-roadmap.md) §4로 승격해 구현합니다.
> 작성 2026-06-23. 코드 앵커(현재 구조): `state`(credits/minerals/player_level/plants[]/nursery_daily/trait_cards/...), `defaultState()`/`normalizeState()`/`saveState()`, `buildEnemy(round,tier)`/`startBattle()`/`startMatch()`/`makeCombatant()`, `activePlant()`/`renderMain()`/`renderCollection()`, `SPECIES{}`/`ELEMENTS{}`/`FORMS{}`, `show()/hide()/toast()/sfx`, 하단 nav `data-nav`(shop/explore/home/nursery/mode).

## 공통 규약(8개 모두 해당)
- **세이브:** 새 필드는 `defaultState()`에 기본값 추가 + `normalizeState()`에 마이그레이션 1줄(`if(!s.xxx) s.xxx = ...`). 기존 세이브 깨짐 0 보장(현재 코드가 쓰는 방식 그대로).
- **시간/일일 리셋:** 이미 있는 패턴 재사용 — `nurseryToday()`('YYYY-MM-DD') vs 저장된 `day` 비교 후 리셋. 일일 퀘스트·출석·방치보상 모두 이 패턴 1개로 통일.
- **UI 진입:** 전체화면은 `#nurseryScreen`처럼 `position:fixed; top:headh; bottom:navh; z-index:50` 오버레이 패턴. 팝업은 `.modal`/`.modal-card` 재사용.
- **사운드/피드백:** `sfx.win/heal/tap`, `toast()` 재사용.

# ① 방치(오프라인) 진행 ⭐ 최우선

> **왜:** 간판이 "방치형"인데 닫으면 아무 일도 안 일어남. AFK류 리텐션의 뿌리.
> **핵심 데이터:** `state.last_seen`(ISO 시각) 저장 → 접속 시 `now - last_seen = 경과초`로 보상 계산. 상한(`OFFLINE_CAP_HOURS`, 예 8h)으로 무한누적 방지.

### 선택지 A — 양육 자동 진행 *(가장 가벼움)*
- **동작:** 닫혀 있는 동안 화분 식물의 물/비료 게이지가 시간에 비례해 자동 충전. (예: 1시간당 물 +2). 단, **일일 한도(`NURSERY_DAILY_WATER/FERT`)는 그대로 적용** → 밸런스 안전.
- **UI:** 접속 시 중앙 모달 "🌙 자리를 비운 사이 — 💧물 +12 🌿비료 +5 자동 공급 / 🍎열매 2개 맺힘" + [받기] 버튼. 양육 탭 상단에 "오프라인 +N" 작은 뱃지.
- **데이터:** `state.last_seen`. 계산은 화분별 `p.nursery.water/fert`에 가산 후 `checkNurseryFruitReady(p)` 호출.
- **구현:** `loadState()` 직후 `applyOfflineProgress()` 신규 함수 1개 → 경과시간×속도, 한도 클램프, 모달 표시. 끝에 `state.last_seen = now; saveState()`. 종료/주기적 저장 시 `last_seen` 갱신.
- **난이도:** ★☆☆ (반나절). 리스크 낮음(기존 양육 함수 재사용).

### 선택지 B — 자원 자동 수급
- **동작:** 시간당 크레딧/미네랄/종자 소량 누적(상한). 함선 가구(로드맵 #4)로 효율 배수.
- **UI:** A와 같은 "자리 비운 사이" 모달에 💰/💠/🌱 라인 추가.
- **데이터:** `OFFLINE_RATE = {credits:N/h, minerals:.., seeds:..}`, 함선 업글 레벨로 배수.
- **구현:** `applyOfflineProgress()`에서 자원 가산. 종자는 `state.seed_inventory.push(...)`.
- **난이도:** ★★☆. 리스크: 경제 인플레 → 상한·요율 보수적으로.

### 선택지 C — A+B 둘 다 (풀 방치 루프) — *추천*
- 모달 1개에 양육+자원 통합 표시. 가장 "방치형다움".

### 선택지 D — 안 함 (수동 유지)

> **수정 포인트 예시:** 충전 속도, 상한 시간, "광고 보고 2배"(추후 수익화) 자리 둘지.

# ② 매일 돌아올 이유 (리텐션 루프)

> **데이터 기반:** ①의 일일 리셋 패턴 공유. `state.daily = { day, quests:{...}, claimed:[] }`.

### 선택지 A — 일일 퀘스트 3종
- **동작:** 매일 자정 리셋되는 과제 3개. 예: ⚔️전투 3승 / 💧물 5회 / 🛰️탐사 1회. 완료 시 개별 보상 + 3개 다 완료 시 보너스(보급상자).

### 선택지 B — 출석 보상(7일 캘린더)

- **UI:** 7칸 그리드 모달(오늘 칸 하이라이트, 받은 칸 체크).
- **데이터:** `state.attendance = { lastDay, streak, claimedDay }`.
- **구현:** `checkAttendance()`(접속 시 1회), `renderAttendance()`, `claimAttendance()`.
- **난이도:** ★☆☆.

### 선택지 D — 안 함

# ③ 엔드게임 — 무한 도전(현재 단판 토너먼트 너머)

> **현 구조:** `startBattle()`→`state.tournament={roundIndex,...}`, 봇은 `buildEnemy(roundIndex, tierIndex)`로 절대스탯 생성(플레이어와 분리). 이걸 그대로 확장하면 됨. 하단 nav 5칸이 꽉 차 있으므로(상점/탐사/홈/양육/함선) **진입은 홈의 토너먼트 바 옆 새 버튼**으로.

### 선택지 A — 무한 콜로세움(로그라이크 런) — *추천*
- **동작:** 라운드를 무한히 올리며 연승. 라운드마다 `buildEnemy(round, tier)`의 round를 +1씩 키워 점점 강해짐. **라운드 사이마다 강화 3중 택1**(예: "공격 +12% / 스킬 1개 등급↑ / 체력 회복") — 이 강화는 **해당 런에만 적용되는 임시 버프**(영구 아님). 패배 시 종료, **도달 라운드에 비례해 메타 재화(예 🌟별가루) 지급** → 영구 강화/잠금해제에 사용.
- **UI:**
  - 홈 토너먼트 바 옆 "♾️ 무한 콜로세움" 버튼.
  - 진입 시 풀스크린 오버레이: 현재 라운드/최고 기록, 누적 임시버프 아이콘 줄, [다음 전투].
  - 라운드 사이 "강화 선택" 모달 = 카드 3장(`.modal-card` + 3개 `.dcard` 스타일 재사용).
  - 패배 화면: "라운드 14 도달! 🌟별가루 28 획득".
- **데이터:** `state.endless = { active, round, buffs:[], best }`, `state.stardust`(메타재화). 임시버프는 전투 진입 시 `makeCombatant`의 baseStats/loadout에 합산하는 `applyRunBuffs(combatant, buffs)`.
- **구현:** `openEndless()`, `endlessNextBattle()`(기존 `startMatch` 재사용하되 종료 콜백을 분기), `rollRunUpgrades()`(택1 3장), `applyRunBuffs()`, `endEndlessRun()`(별가루 지급). 영구 강화 상점(별가루 소비)은 후속.
- **난이도:** ★★★ (전투 종료 콜백 분기가 핵심). 리스크: 토너먼트 흐름과 상태 충돌 → `state.tournament`와 `state.endless`를 **상호배타 모드 플래그**로 분리.

### 선택지 B — 일일 시련
- **동작:** 매일 바뀌는 특수 규칙 단판(예: "모든 화상 2배", "에너지 시작 +2"). 등수/클리어 보상.
- **UI:** 홈 버튼 → 오늘의 규칙 카드 + [도전]. 클리어 시 도장.
- **데이터:** `state.trial = { day, cleared }`. 규칙은 시드(날짜) 기반 결정.
- **구현:** `todaysTrial()`(날짜 시드로 규칙 선택), 전투 modifier 훅을 `startMatch`에 주입.
- **난이도:** ★★☆.

### 선택지 C — A+B 둘 다
### 선택지 D — 다른 것(보스 레이드 등) — 메모로

> **수정 포인트:** 메타재화 이름/용도, 임시버프 풀, 영구강화 상점을 같이 만들지(추천: 1차엔 별가루만 적립, 상점은 2차).

# ④ 식물 번식 / 교배 (테마 강화)

> **현 구조:** 식물 생성은 `createPlantedPlantFromSeed(entry, seed, name)`·`createPlant(species, alloc)`로 `state.plants.push(p)`. 속성/변이형/잠재력 필드 존재. 양육 탭(`#nurseryScreen`)에 자연스럽게 붙음.

### 선택지 A — 2종 교배 → 새 종자 — *추천*
- **동작:** 화분 식물 2마리 선택 → 일정 시간/자원 후 **자식 종자** 1개 산출. 자식은 부모의 속성/변이형/잠재력을 **확률 유전**(예: 속성 50:50, 변이형 우월 상속, 잠재력은 부모 평균±1, 소확률 상위 돌연변이). 결과는 `state.seed_inventory`로.
- **UI:** 양육 탭에 "💞 교배" 버튼 → 부모 2칸 선택 슬롯 + 미리보기(가능 결과 속성/변이 후보) + [교배 시작]. 진행 중엔 화분처럼 타이머(또는 ① 방치 채택 시 오프라인 진행).
- **데이터:** `state.breeding = { slotA, slotB, startedAt, ready }`. 유전 규칙 `breedOffspring(pa, pb)` 순수함수.
- **구현:** `openBreeding()`, `pickBreedParent()`, `breedOffspring()`, `claimOffspringSeed()`(→ seed_inventory). 비용은 크레딧+미네랄 또는 열매.
- **난이도:** ★★★ (유전 규칙 설계가 핵심). 리스크: 무한 자가증식 → 쿨다운/비용/세대 제한.

### 선택지 B — 열매 → 씨앗 회수
- **동작:** 현재 열매(스킬 박스)에 더해, 수확 시 "🌰 씨앗 맺기" 분기 → 같은 종 종자(소확률 변이) 산출. 가장 가벼운 번식.
- **UI:** `doHarvestPot` 결과 모달에 [스킬로] / [씨앗으로] 선택.
- **데이터:** 기존 `nursery_fruits` 흐름 재사용 + `seed_inventory` 분기.
- **구현:** `doHarvestPot()` 분기 + `fruitToSeed(p)`.
- **난이도:** ★☆☆.

### 선택지 D — 안 함

# ⑤ 전투 편성 — 1마리 → ?

> **현 구조:** `activePlant()` 1마리만 출전(`startBattle`). 가장 구조적 변경이라 신중.

### 선택지 A — 현행 유지(1마리) *(안전)*
- 클래시로얄식 단일 영웅 정체성 유지. 다른 항목에 자원 집중.

### 선택지 B — 2~3마리 파티 + 교체
- **동작:** 출전 파티(최대 3) 편성. 전투 중 턴 소모로 식물 교체(속성 상성 운영). 한 마리 기절 시 다음 교체, 전멸 시 패배.
- **UI:** 홈에 "파티 편성" 슬롯 3칸(현재 컬렉션 바 활용). 전투 화면에 하단 벤치 아이콘 + [교체] 액션.
- **데이터:** `state.party = [id1,id2,id3]`. 전투 엔진은 현재 `me` 1명 → `meTeam[]`+activeIndex로 확장.
- **구현:** `startMatch`/턴 루프/`makeCombatant`를 팀 단위로 리팩터. **가장 큰 작업**.
- **난이도:** ★★★★. 리스크: 전투 엔진 전반 수정 → 회귀 위험. 단계적(먼저 데이터·UI, 그다음 엔진).

### 선택지 C — 벤치 패시브(절충)
- **동작:** 전투엔 1마리, 벤치 식물이 **출전 식물에 패시브 버프**만 제공(예: 벤치 풀속성 1마리당 회복 +3%). 엔진 변경 최소.
- **UI:** 편성 슬롯 + "응원 효과" 요약.
- **데이터:** `state.party` 동일, 버프는 `calcStats`/전투 시작 시 합산.
- **구현:** `benchBonuses()` → 출전 스탯에 가산. 엔진 구조 유지.
- **난이도:** ★★☆.

> **수정 포인트:** 파티 인원(2/3), 교체 비용(턴/에너지), 벤치 버프 종류·상한.

# ⑥ 수집 도감 & 완성 보상

> **현 구조 주의:** 지금 `#collectionBar`/`renderCollection()`는 **도감이 아니라 보유 식물 로스터**(화분 칩). 진짜 "도감"은 신규. 종 데이터는 `SPECIES{}` 존재(이름/속성). 변이카드는 `card_*` 정의 + `state.trait_cards{}`.

### 선택지 A — 종 도감
- **동작:** 한 번이라도 보유/획득한 `SPECIES`를 "발견"으로 기록. 종별 발견 시 소량 영구 보상(예 💠미네랄), 발견률 25/50/75/100%에 마일스톤 보상(보급상자/칭호). 미발견은 실루엣.
- **UI:** 새 풀스크린(또는 설정 옆 "📖") → 속성별 탭 + 그리드 카드(발견=컬러/스프라이트, 미발견=실루엣+❔). 상단에 도감 달성률 바.
- **데이터:** `state.dex = { species:{[id]:true}, milestones:[] }`. 식물 생성 지점에서 `markDexSpecies(sp)`.
- **구현:** `markDexSpecies()`(생성 함수들에 1줄), `openDex()`, `renderDex()`, `claimDexMilestone()`.
- **난이도:** ★★☆.

### 선택지 B — 변이카드 도감
- **동작:** 보유/획득한 `card_*`를 등급별로 수집 현황 표시. 세트 완성 보상.
- **데이터:** 기존 `state.trait_cards` 키 + "한 번이라도 가졌던" 집합 `state.card_seen`.

# ⑦ PvP

### 선택지 A — 비동기 고스트 PvP — *추천*
- **동작:** 내 대표 식물의 **로드아웃 스냅샷**을 서버(또는 공유 풀)에 저장. 상대의 스냅샷을 내려받아 **봇 AI가 재생**해 대전. 실시간 동기화 불필요 → 서버 부담 최소. 승패로 PvP 점수/주간 랭킹.
- **UI:** "🏟️ PvP" 진입 → 상대 카드 3개 중 택1 → 전투(기존 엔진 재사용, 적을 `buildEnemyFromSnapshot()`로). 주간 랭킹 보드.
- **데이터:** 스냅샷 = `{species, element, form, growth, baseStats, loadout, skillGrades, cards, ownerName, score}`. 저장은 현재 쓰는 **Supabase 클라우드 세이브** 인프라 재사용(테이블 `pvp_ghosts`).
- **구현:** `makeCombatant`가 스냅샷을 그대로 먹으므로 적 생성은 쉬움. `uploadGhost()`/`fetchGhosts()`(Supabase), `openPvp()`, 점수 갱신.
- **난이도:** ★★★ (네트워크 파트). 리스크: 오프라인 폴백(고스트 못 받으면 봇 대체).

### 선택지 B — 실시간 PvP
- 매치메이킹·동기화 서버 필요. **후순위 권장.**
- **난이도:** ★★★★★.

# ⑧ 편의성(QoL) — *전부 채택 요청하심 → 8개 모두 설계*

> 작고 독립적. 회귀 위험 낮음. 묶어서 한 번에 또는 순차 적용.

### ⑧-1 자동 전투
- **동작:** 전투 중 [AUTO] 토글 → AI가 매 턴 스킬 자동 선택(현 봇 AI 로직 재사용).
- **UI:** 전투 화면 우상단 토글 버튼(켜짐=강조). 상태 `state.ui.autoBattle`.
- **구현:** 플레이어 턴에 autoBattle이면 봇 `chooseSkill()`을 플레이어 측에 적용 + 짧은 딜레이.
- **난이도:** ★☆☆.

### ⑧-2 전투 배속 (×1/×2/×3)
- **동작:** 애니메이션/딜레이 타이머에 배속 계수.
- **UI:** 전투 화면 "⏩ ×2" 순환 버튼. `state.ui.battleSpeed`.
- **구현:** 전투 연출 `setTimeout(ms)`들을 `ms/speed`로. (타이머 상수 한 곳에 모아 계수 적용 권장.)
- **난이도:** ★★☆(타이머 산재 시 정리 필요).

### ⑧-3 양육 "전체 물주기/비료"
- **동작:** 양육 탭 상단 [💧전체] [🌿전체] → 일일 한도 내에서 모든 화분에 1회씩 일괄.
- **UI:** `.nursery-top`에 버튼 2개.
- **구현:** `waterAll()`/`fertAll()` = 화분 순회하며 `doWaterPot` 로직, 한도 소진 시 중단 + 요약 토스트.
- **난이도:** ★☆☆.

### ⑧-4 보급상자 10연속 개봉 + 결과 요약
- **동작:** 상점 보급상자 [×10] 구매·개봉 → 결과 리스트 모달(등급별 정렬).
- **UI:** 상점 상자 카드에 [10連] 버튼, 결과 그리드 모달.
- **구현:** 기존 단일 개봉 함수 N회 루프 + 결과 수집 → `renderOpenSummary()`.
- **난이도:** ★★☆.

### ⑧-5 식물 목록 정렬/필터
- **동작:** 보유 식물 목록을 속성/등급/생장단계/잠재력로 정렬·필터.
- **UI:** 컬렉션/식물 선택 화면 상단 칩 필터 + 정렬 드롭다운.
- **구현:** `sortPlants(mode)`/`filterPlants(pred)` → `renderCollection` 입력에 적용. 상태 `state.ui.plantSort`.
- **난이도:** ★★☆.

### ⑧-6 전투 결과 스킵/빠른 진행
- **동작:** 판정창·연출을 탭 한 번에 스킵, "결과만 보기" 옵션.
