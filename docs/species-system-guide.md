# 풀로세움 — 식물 종(속성 × 타입) 공략집

> **핵심:** 모든 식물의 기본 스탯 = **속성 수치 + 타입 수치**.
> 속성(7종)은 상성·색·전투 성향을, 타입(5종)은 몸체(실루엣)·체격 성향을 담당한다.
> **5타입 × 7속성 = 총 35종.** 생장단계별 외형은 절차적으로 자동 생성된다.
>
> ⚠️ **개체 카탈로그 전환(2026-06-24):** 종 시스템이 자동 격자에서 **수작업 개체 카탈로그(`SPECIES_CATALOG`) + 스킬 정의 분리(`SKILL_LIB`)** 로 전환됐다. 같은 타입·속성·단계여도 스킬·변이 슬롯이 다른 별개 개체를 추가할 수 있고, 종별 **희귀도(`rarity`)** 로 등장/드롭률을 차등한다. 설계서·계획서: [`superpowers/specs/2026-06-24-plant-individual-catalog-design.md`](superpowers/specs/2026-06-24-plant-individual-catalog-design.md) · [`superpowers/plans/2026-06-24-plant-individual-catalog.md`](superpowers/plans/2026-06-24-plant-individual-catalog.md)
> **타입 개편:** 5타입 = **버섯 / 목본 / 화초 / 다육 / 덩굴**. 버섯형은 저스탯+포자 기본+희귀.
> **초본형 폐지(2026-06-24):** 초본형(grass 타입)은 "풀은 결국 꽃을 피운다"는 이유로 **화초형으로 완전 흡수**. (구)초본 7종은 타입이 `flower`로 전환(스탯·외형·스킬 모두 화초형), 세이브의 `seed_type:'grass'`도 `flower`로 마이그레이션. ⚠️ `grass`는 이제 **풀 '속성'으로만** 존재(타입 아님). 내부 `TYPE_STATS.grass`는 안전망 폴백으로만 잔존(미표시).
>
> 기준일: 2026-06-24(개편) · 출처: `index.html` (`ELEMENT_STATS` / `TYPE_STATS` / `SPECIES_GRID` / `SPECIES_CATALOG` / `SPECIES` / `SKILL_LIB`)

## 4. 탐사·획득·키우기·매칭 연동

35종은 게임 전 흐름에 연결돼 있다.

- **탐사 발견(아틀라스 종 분포, 2026-06-24~):** 탐사 성공 시 종자는 `rollSpeciesFromView(region, planet)`로 결정된다 — **행성 서식 풀(`planet.species`) ∩ 지역 테마(속성 `el` · 타입 `types`)** 에서 희귀도 가중 추출. 즉 행성마다 "사는 종"이 정해지고, 같은 행성 안 지역끼리는 속성/타입으로 갈린다. 추가로 지역마다 **`signature`**(풀에서 제외한 희귀 전용종)가 `SIGNATURE_CHANCE`(0.10) 확률로 따로 등장(포켓몬식 "특정 서식지 희귀종"). 신규 필드가 없는 행성/지역은 기존 테마 전역 동작으로 폴백(무회귀). 종자 엔트리에 `species`/`element`/`seed_type`가 기록된다. → 데이터: `index.html`의 `EXPLORE_VIEW`(행성 `species`·지역 `signature`).
- **종자 가방:** 카드·상세에 결정된 종의 타입·종 이름이 표시된다.
- **키우기(심기):** `createPlantedPlantFromSeed`가 종자의 `species`를 그대로 식물에 적용 → 외형 타입·속성·스탯이 종에 맞게 고정.
- **매칭(전투):** 적은 35종 전체에서 무작위 출현(`buildEnemy`). 라운드·티어로 생장단계·등급이 오른다.

## 5. 이미지(외형) — 생장단계별 자동 생성

식물 외형은 그림 파일이 아니라 **절차적 SVG**로 만들어진다(`composePlantBody` / `composePlantSvg`).
- **입력:** 타입(실루엣) × 속성(색·모티프) × 생장단계(크기).
- **생장 6단계:** 씨앗 → 새싹 → 유체 → 성장체 → 성체 → 완숙체. 단계가 오를수록 실루엣이 자라고, 완숙체엔 오라+✦가 붙는다.
- 따라서 **35종 × 6단계 = 210개 외형이 전부 자동 지원**된다. 별도 PNG 추가 불필요.
- 특정 조합만 손그림으로 교체하려면 `SPRITE_OVERRIDES[`타입_단계_속성`]`에 이미지 URL을 넣으면 된다.

타입별 실루엣 요지:
| 타입 | 실루엣 |
|---|---|
| 🌿 초본형 | 여러 갈래 잎날이 부채처럼 펼쳐짐 |
| 🌳 목본형 | 줄기 위 둥근 수관(나무) |
| 🌸 화초형 | 봉오리→개화(꽃잎 방사) |
| 🌵 다육형 | 통통한 기둥 + 가시·곁가지 |
| 🍃 덩굴형 | 휘감기는 덩굴 줄기 + 작은 잎 |
| 🍄 버섯형 | 통통한 자루 + 점박이 갓 + 고리(완숙=포자 입자) |

### 타입별 생장 단계명 (`STAGE_NAMES_BY_TYPE`)

> **2026-06-24 신설.** 기존 "씨앗→새싹→…" 공통 단계명을 타입별로 차별화 → 세계관 몰입·타입 정체성 강화. 코드: `STAGE_NAMES_BY_TYPE` / `growthStageName(stage, seedType)`.

| 단계(코드) | 목본(tree) | 화초(flower) | 다육(cactus) | 덩굴(vine) | 버섯(mushroom) |
|---|---|---|---|---|---|
| seed | 씨앗 | 씨앗 | 씨앗 | 씨앗 | 포자 |
| sprout | 떡잎 | 새싹 | 움 | 새순 | 균사 |
| juvenile | 묘목 | 새잎 | 어린기둥 | 어린덩굴 | 버섯눈 |
| growing | 어린나무 | 꽃봉오리 | 가시몸 | 덩굴손 | 어린갓 |
| mature | 성목 | 개화 | 성체선인장 | 성체덩굴 | 성숙버섯 |
| evolved | 거목 | 만개 | 꽃선인장 | 만연덩굴 | 포자갓 |

## 6. 개발자용 — 수치를 만질 때 (index.html)

| 만지고 싶은 것 | 상수/함수 |
|---|---|
| 속성별 스탯 기여 | `ELEMENT_STATS` |
| 타입별 스탯 기여 | `TYPE_STATS` |
| 종 목록(이름·타입·속성) | `SPECIES_GRID`(레거시 35종) + `SPECIES_CATALOG`(개체 확장) → `SPECIES` 자동 머지 |
| 개체별 rarity/변이슬롯/태생변이/단계스킬/시그니처 | `SPECIES_CATALOG[key]` |
| 신규/버섯/시그니처 스킬 정의(분리) | `SKILL_LIB` (→ `ALL_SKILLS`에 합산) |
| 희귀도 가중 획득(legacy 제외) | `RARITY_WEIGHT` / `pickAcquirableSpecies(filterFn?)` |
| 변이 슬롯·태생 변이 식물 적용 | `applyCatalogVariantFields(p)` (심기·세이브 마이그레이션 공용) |
| 속성 상성 배율 | `ELEMENTS[].strong` / `eff()` |
| 종→외형 타입 매핑 | `SPECIES_SEEDTYPE`(SPECIES에서 자동 파생) |
| 타입×속성 → 종 역참조 | `SPECIES_BY_TYPE_EL` / `speciesForTypeEl()` |
| 탐사 지역 테마 → 종 | `rollSpeciesFromView()` |
| 종자 엔트리 생성 | `createExplorationSeedEntry(seed,…,speciesKey)` |
| 종자 → 식물 심기 | `createPlantedPlantFromSeed()` |
| 무지개 시작 종자 | `makeRainbowStarterSeed()` / `claimStarterSeed()` |
| 외형 생성 | `composePlantBody()` / `composePlantSvg()` |
| 손그림 교체 | `SPRITE_OVERRIDES` |

> 종을 추가/수정하려면 **`SPECIES_GRID`에 `[key, 이름, 타입, 속성]` 한 줄**만 넣으면 스탯·외형·적 등장·탐사·상성이 자동 연결된다. (기존 7종 key: flame/aqua/thorn/gaia/windy/spark/frost — 세이브 호환 위해 유지)
