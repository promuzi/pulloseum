# 식물 타입·변이형·잠재특성 로드맵

> 상태 업데이트(2026-06-16): 용어 체계를 재정리했다. `index.html`의 기존 저장 키(`form`, `traits`, `trait_cards`)는 세이브 호환을 위해 유지하지만, 화면과 기획 명칭은 아래 기준을 따른다.

## 1. 확정 용어

- **식물 타입**: 식물 외형과 이름의 기본 축. 기존 풀/나무/꽃/선인장/덩굴은 각각 **초본형/목본형/화초형/다육형/덩굴형**으로 부른다.
- **속성**: 불/물/풀/대지/바람/번개/빙결. 전투 상성과 속성 스킬을 담당한다.
- **생장 단계**: 씨앗→새싹→유체→성장체→성체→완숙체. 성장 경험치로 진행되며 스탯 배수와 스킬 해금 곡선의 기준이다.
- **변이형**: 일반/포식/무기/독성/포자/발광/용족. 변이 카드 장착 가능 범위를 정한다. (개체 카탈로그 전환 후 **개체별로 변이 슬롯 배분이 다르고, 1개 이상 보유**할 수 있다 — 아래 §6 참조.)
- **잠재특성**: 생장 단계에서 나중에 발현될 패시브의 분류. 현재는 종자에 후보로만 표시하고 전투, 스탯, 스킬에는 반영하지 않는다.

## 2. 현재 구현

- **식물 타입 5종**: `SEED_TYPE_NAMES`, `SEED_TYPE_OF`, `seedTypeOf()`.
- **속성 7종**: `ELEMENTS`, `eff()`.
- **생장 단계 6단계**: `GROWTH_STAGE_ORDER`, `GROWTH_STAGE_NAMES`.
- **변이형 7종**: `FORMS`, `rollForm()`.
- **변이 카드 장착**: `TRAIT_CARDS`, `CARD_SKILLS`, `formCardPool()`, `cardEffects()`.
- **외형 자동 생성**: 식물 타입 × 생장 단계 × 속성 조합으로 `composePlantSvg()`, `spriteFor()`가 생성.
- **이름 자동 생성**: 속성 + 장착 변이 카드 + 식물 타입을 `composePlantName()`, `plantDisplayName()`에서 조합.

## 3. 변이형과 카드

| 변이형 | 장착 카드 | 현재 효과 |
|---|---|---|
| 일반 변이형 | 공통 변이 | 기본 방어/반사 계열 카드 |
| 포식 변이형 | DNA | 포식 기본 스킬 보유, DNA 카드로 포식 강화 |
| 무기 변이형 | 무기 | 장착 카드가 무기 스킬을 자동 추가 |
| 독성 변이형 | 물약 | 독성 스킬/독 계수 카드 |
| 포자 변이형 | 포자 | 턴 종료 시 확률 발동 |
| 발광 변이형 | 엽록체 | 에너지/체력 회복 |
| 용족 변이형 | 용 DNA | 미설계 |

## 4. 잠재특성 방향

잠재특성은 삭제하지 않는다. 다만 현재는 **종자 상세의 후보 정보**로만 남긴다.

예시:
- **과성장**: 이후 생장 단계에서 레벨 없이 기본 스탯 상승폭이 커지는 패시브.
- **안정 코어**: 특정 생장 단계부터 안정도 하락을 완화하는 패시브.
- **적응 게놈**: 환경별 성장 보정이나 변이 안정화에 관여하는 패시브.

아직 전투나 스탯 계산에는 연결하지 않는다. 구현 시점에는 `possible_traits` 후보, 생장 단계, 발현 조건, 패시브 효과 테이블을 따로 설계한다.

## 5. 남은 작업

- 잠재특성 발현 조건과 패시브 효과 설계.
- 생장 단계별 스킬 해금 곡선 구체화.
- 용족 변이형 카드 설계.
- 식물 타입별 스탯 성향 정의.
- 커스텀 외형 오버라이드 확장.

## 6. 외형 오버라이드 키

키 형식: `"<식물타입>_<생장단계>_<속성>"`

- 식물타입: `grass(레거시)|tree|flower|cactus|vine|mushroom`
- 생장단계: `seed|sprout|juvenile|growing|mature|evolved`
- 속성: `fire|water|grass|earth|wind|bolt|ice`

예: `grass_juvenile_fire`

## 7. 개체 카탈로그: 변이 슬롯 · 시그니처 · 스킬 분리 (2026-06-24)

종 시스템이 **수작업 개체 카탈로그**로 전환됐다. 같은 타입·속성·단계여도 스킬/변이 슬롯이 다른 별개 개체를 만들어 수를 늘린다. (설계서: [`superpowers/specs/2026-06-24-plant-individual-catalog-design.md`](superpowers/specs/2026-06-24-plant-individual-catalog-design.md))

- **개체 정의(`SPECIES_CATALOG[key]`)**: `{ name?, type?, element?, rarity?, stats?, baseVariants?[], variantSlots?{변이형:칸수}, signatures?[], stageSkills?{단계:[스킬키]}, legacy? }`. 없는 필드는 자동 기본값.
- **변이 슬롯(`variantSlots`)**: 개체마다 변이형별 장착 칸 배분이 다르다(예 `{spore:2, normal:4}` = 2:4). 변이를 1개 이상 보유할 수 있는 근거. (슬롯 편집 UI는 향후 작업 — 현재는 식물에 데이터만 기록.)
- **태생 변이(`baseVariants`)**: 태어날 때부터 갖는 변이형. 버섯 개체는 `['spore']` 고정. **그 외 종은 비움** — "포자=버섯 전용"을 하드코딩하지 않고 데이터로만 결정해, 나중에 다른 종에 `'spore'` 한 줄 추가하면 확장된다.
- **시그니처(`signatures`)**: 개체 고유 스킬 키. 변이 연동 고유기(예: 무기 화염화)를 여기에 둔다.
- **단계 스킬(`stageSkills`)**: 생장 단계별 해금 스킬. 공통(속성/단계) 위에 **개체 고유 비중을 크게**. 카탈로그에 `stageSkills`가 있으면 그 경로, 없으면 레거시 `ELEMENT_GROWTH_SKILLS` 풀을 쓴다(`plantKnownSkillIds`).
- **스킬 정의 분리(`SKILL_LIB`)**: 신규/버섯/시그니처 스킬의 실제 효과는 `SKILL_LIB` 한 곳에만 정의하고 개체는 키로만 참조한다(`ALL_SKILLS`에 합산). 네임스페이스: `common.<속성>.* / sig.* / mushroom.*`. **향후 스킬 추가·수정은 여기와 카탈로그만 건드린다.**
- **희귀도(`rarity`)**: 등장/드롭률만 차등(`RARITY_WEIGHT`), 스탯 차이는 작게. 희귀도↑ → 시그니처/변이 다수. 개체 내 품질 차이는 없음.
