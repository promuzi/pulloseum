# 설계: 개체 고유화 — 타입/속성 공통 컨셉 + 개체별 스토리·고유 스킬·외형

작성일 2026-06-24 · 상태: **설계 단계 — 코드 미반영(소유권 대기)**

> **재개 지점:** 공통 레이어(타입5·속성7·변이 재편)·개체 35종(3스킬)·변이 개체 샘플 3종·화분/식물 분리·외형 접근까지 **설계 확정**. 변이 개체 1차 어울림 28종(V1~V28)도 완성. **남은 설계**=각 종 2차 어울림 변이 + 외형 액센트 명세(아래 "남은 설계"). **코드 반영**은 버섯/변이 세션이 같은 `index.html`을 작업 중이라 **소유권 정해진 뒤 일괄**.

## 목표
포켓몬처럼 **각 개체가 확실한 스토리·컨셉**을 갖게 한다. 공통으로 묶이는 가벼운 레이어(타입·속성)를 참고선으로 두고, **개체 하나하나에 고유 스토리 + 고유 스킬 3개 + 외형**을 부여한다. 사용자와 한 마리씩 협업 설계 → 승인되면 이 문서에 누적 → 나중에 코드 일괄 반영.

## 범위 결정
- **(타입×속성) 한 칸 = "계열"**, 한 칸에 **여러 개체** 가능(포켓몬식). 지금은 각 칸의 **첫 개체(기둥 개체)** 부터 채우고, 계열 추가 개체는 나중에 확장. (= 처음엔 1칸 1개체로 시작하되 구조는 다개체 허용)
- 코드 구조는 이미 다개체 허용: `SPECIES_CATALOG`(개체 키 자유) + `SKILL_LIB`(스킬 정의). 영구 결정 불필요.

## 대원칙 — 타입·속성은 "기본값", 개체가 깬다
- 타입 한 줄·속성 한 줄은 **강제가 아니라 참고**. 같은 불이라도 공격형만 있는 게 아니라 "치유의 온기" 회복형도, 같은 목본이라도 공격형 나무도 가능.
- **창작 가이드**: 개체·스킬을 만들 때 한 속성/타입을 한 색으로만 찍지 말고 **유연하게 변주**한다.
- 단순화: 스탯은 `TYPE_STATS + ELEMENT_STATS` 그대로(개체별 스탯 변경 없음). "깸"은 **컨셉·고유 스킬**로 표현(별도 스탯 배관 없음).

## 공통 레이어 1 — 타입(스킬 참고용 한 줄)
| 타입 | 역할 | 시그니처 키워드 |
|---|---|---|
| 🌳 목본 | 탱커 | **버팀 & 재생** (방어·힐·에너지 충전) |
| 🌸 화초 | 화력 | **개화** (공격 버프 누적 → 폭발) |
| 🌵 다육 | 벽 | **반격 & 저장** (counter·방어/힐 비축) |
| 🍃 덩굴 | 기동 | **흡혈 & 속도** (lifesteal·선공) |
| 🍄 버섯 | 교란 | **포자** (중독·약화 등 상태이상) |

## 공통 레이어 2 — 속성(스킬 참고용 한 줄)
| 속성 | 성향 | 상성(강점 / 약점) | 상태이상 테마 |
|---|---|---|---|
| 🔥 불 | 고공격/유리 | 풀·빙결 / 물 | **화상**(지속딜) |
| 💧 물 | 견고/방어 | 불·대지 / 풀·번개·빙결 | **회복** |
| 🌿 풀 | 균형 | 물·대지 / 불·바람·빙결 | **공격 버프** |
| 🪨 대지 | 중장갑/둔중 | 번개·바람 / 풀·물 | **방어 버프** |
| 🌪️ 바람 | 최속 | 풀 / 대지 | **기동 버프** |
| ⚡ 번개 | 유리대포 | 물 / 대지 | **치명타** |
| ❄️ 빙결 | 지구력 | 물·풀 / 불 | **둔화** |

> 속성도 유연: 위 테마는 기본색일 뿐, 개체가 다른 계열(예: 불+회복)을 가질 수 있다.

## 개체 템플릿 (한 마리마다)
- **이름 / key**
- **타입 · 속성** (스탯 자동 = 타입+속성)
- **희귀도**
- **컨셉/스토리** (1~2줄, 포켓몬식 정체성)
- **고유 스킬 3개(이상)** — **성장체 / 성체 / 완숙체 각 1개**(상위 단계 정체성). 필요하면 더.
- **외형 메모** (절차적 SVG에 얹을 특징, 나중에 PNG)

### 고유 스킬 규칙
- **새싹·유체** = 기존 공유 스킬(`STAGE_SKILLS`) 그대로. **시그니처는 성장체부터** 해금.
- 상위 3단계(성장체·성체·완숙체)에 1개씩 → 비어 있던 레거시 `ELEMENT_GROWTH_SKILLS` 구간을 **개체 고유 스킬로 대체**(개체 정체성의 핵심).
- **엔진 제약(결정형, `applyMove` 지원 필드만):** 한 스킬은 `guardMult(+counterPct)`·`energyGain`·`cleanse`·`heal`·`selfBuff{stat,pct,turns}`를 동시에 가질 수 있고, `power>0`이면 공격분기(`aoe`/`single`·`pierce`·`critBonus`·`lifesteal`·`enemyDebuff{stat,pct,turns}`·`dot{kind,pct,turns}`·`elem`)가 추가로 돈다. 확률성 라이더 없음.

## 변이형 재편 (2026-06-24 확정)
> ⚠️ **변이 시스템 권위 문서 = [mutation-forms-cards-redesign-design.md](2026-06-24-mutation-forms-cards-redesign-design.md)** (다른 세션이 **이미 구현**: 6변이형·발광 폐지·포자=버섯전용·**변이형=패시브 없는 카드 슬롯 게이트**·**식물당 변이형 1개**·카드 정의). 아래는 그 문서와 정합하며, **이 "변이 개체" 작업 = 그 문서가 예고한 "개체-변이 분류(미래)"의 구현**이다.
> **정합 제약(그 문서 §개체-변이 분류):** 개체 **시그니처(고유) ≠ 변이 카드(범용)** — 같은 효과 중복 금지. 변이 개체의 3스킬은 *그 개체만의 결*로(카드가 주는 범용 효과를 그대로 베끼지 않게). 구현 시 카드 효과와 대조 검토.
> 변이 **스킬은 변이 카드로 부여**(종 시그니처와 별개). 개체 고유 스킬엔 변이 카드 스킬 안 넣음.

- **일반 변이 5개** (아무 종에나 카드로 부여 가능):
  1. **일반/공통** — 범용 강화(스탯·에너지·재생). 폐지된 발광(엽록체)의 에너지·HP 재생을 여기로 흡수.
  2. **포식(DNA)** — 흡혈·능력치 흡수(공격하며 약탈·회복).
  3. **무기** — 검·방패·도끼·총 물리 무장 **+ 속성 인챈트(신규 갈래)**.
  4. **독성(물약)** — 독·산·방깎(지속 피해·약화).
  5. **용족** — 용 DNA, 고위력 희귀 프리미엄.
- **포자 = 버섯 타입 전용(고유)** — 일반 변이 풀에서 분리. 버섯에게만(매 턴 포자: 마비·중독·교란). 버섯 세션과 조율.
- **발광(엽록체) = 폐지** — 회복/에너지는 일반 강화로 흡수.
- ⚠️ **구현 파급(나중에 일괄):** `FORMS`·`TRAIT_CARDS`(chloro 제거)·`box_card_*` 보급상자·탐사 `cardTypes`·`cardFitsForm`·UI 변이 탭·버섯 `baseVariants:['spore']`. 지금은 설계 확정만.

### 🗡️ 무기형 속성 인챈트 (별도 트랙)
- 무기형 카드에 **속성 부여 계열**(불/물/번개/빙결) 추가 → 상성을 골라 때리는 무기형만의 전술.
- **(a) 일회성 속성 타격**(엔진 `elem` 필드 그대로, 새 배관 0) 우선 / (b) 지속 인챈트(imbue 상태 새 배관)는 후속.
- 종 작업과 분리된 변이 카드 풀 확장으로 관리.

## 식물 / 화분 분리 설계 (#12·#2·#3 전제)
> 화분을 **따로 장착**(양육)하려면 외형에서 식물과 화분을 분리해야 한다.
- **식물 레이어** = `composePlantBody(seedType, gi, P, el)` (이미 화분 없음 — 그대로 사용).
- **화분 레이어** = 신규 `composePotSvg(potId, size)` — 장착 화분(`p.pot`, 없으면 기본)별로 렌더.
- **합성** = `composePlantSvg`가 *화분(뒤) → 식물(화분 입구 y앵커에 base 정렬)* 2레이어로 스택. 고정 **화분 입구 앵커 y**(현재 화분 rect 위치 ≈ viewBox y96) 기준.
- **idle 흔들림 = 식물 레이어 그룹만** transform(화분 고정).
- viewBox·앵커 동일 유지 → **기존 렌더 무회귀**. 버섯 세션 `bodyStyle` 훅(식물 몸체 변형)과 **직교**(화분은 별개 레이어).
- 데이터: 식물에 `pot` 필드(장착 화분 id). 화분 카탈로그(`POTS`)는 양육(#2)에서 정의 — 전투 스탯 무영향(양육 전용).
- 후속(#3): 식물·화분 각각 `SPRITE_OVERRIDES`로 PNG 교체 가능(레이어가 분리돼 있어 독립 교체).

## 외형 접근 (개체별 특징 반영)
- **지금 단계 = 절차적 SVG 액센트**: 각 개체의 "외형 훅"(로스터에 기록)을 생성기에 per-species 스타일로 반영. 종 데이터에 선택 필드(예: `bodyAccent`/버섯 세션의 `bodyStyle` 확장)로 분기 → 같은 타입·속성이라도 개체 특징(엠버오크의 균열 잉걸 vs 카르노오크의 아가리 옹이 등)이 드러남.
- **도트/PNG 손그림 = #3 픽셀아트 트랙**(전용 AI 도구 필요, 별개 큰 작업). 분리된 식물/화분 레이어 위에 `SPRITE_OVERRIDES`로 얹음.

## 남은 설계(나중에) / 구현 메모 (코드는 소유권 정해진 뒤 일괄)
**남은 설계(안 겹침 — 언제든 가능):**
- 각 종 **2차 어울림 변이 개체**(절반 보류분) + **버섯 비포자 변이**(버섯의 2차 어울림).
- 각 개체 **외형 액센트 명세**(절차적 SVG 파라미터).

**구현(⛔ index.html — 버섯/변이 세션과 충돌, 소유권 정해진 뒤):**
- 개체 = `SPECIES_CATALOG[key]`에 컨셉(desc)·rarity·`signatures`(또는 stageSkills) 추가, 스킬 정의는 `SKILL_LIB`.
- ⚠️ **양육 세션이 같은 `index.html`을 동시 수정 중** → 이 설계는 문서로만 진행하고, 코드 반영은 충돌 안 나게 양육 작업 일단락 후 묶어서.
- 도감은 `__DEX_API`로 라이브 연동이라, 코드 반영 시 도감에 자동 표시(성장체+ 고유 스킬·"개체 고유" 칩).
