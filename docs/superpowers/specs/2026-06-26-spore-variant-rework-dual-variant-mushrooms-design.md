# 포자 변이 재정의 + 이중 변이 버섯 — 설계 (브레인스토밍 중, 재개 예정)

> 작성 2026-06-26. **미완 설계 — 방향만 합의, 나중에 다시 고민·구상·구현.** 재개 시 이 문서부터.
> 관련: [변이형·카드 재설계](2026-06-24-mutation-forms-cards-redesign-design.md) · [변이 개체 정체성](2026-06-26-variant-individual-identity-design.md) · [버섯 타입 설계](2026-06-24-mushroom-type-design.md)

## 목표
버섯의 양을 늘린다. 방법 = **처음으로 변이를 2개 가진 종**을 만든다. 즉 기존 "포자 단일 변이 버섯" 위에 **포자 + (두 번째 변이)** 이중 변이 버섯을 추가한다. 단, "포자 스킬 묶음 + 다른 변이 스킬 묶음"을 한 종에 욱여넣어 **스킬 수만 늘리는** 방식은 거부 — 깔끔하게 공존하는 구조로 간다.

## 핵심 합의 (이번 세션에서 확정한 것)

### 0. 현재 시스템 사실관계 (조사 결과)
- **변이 카드 슬롯 = 총 5칸** (`equipped_trait_cards.slice(0,5)`, `index.html` ~6550). **칸별 예약 없음.** 5칸 안에 "장착 가능한 카드"면 무엇이든 채움.
- 장착 가능 여부 게이트 = **`cardFitsForm(key, form)`** (~5193): `common` 카드는 form이 dragon만 아니면 OK / 그 외 카드는 카드 타입이 form의 `cardType`와 일치해야 OK.
- `variantSlots:{spore:2,normal:4}` 객체는 **실제 장착 제한에 안 쓰임** — 표시·백필용 잔재. (진짜 게이트 = `cardFitsForm` + 총 5칸)
- `p.form`은 **단수**. `p.form = p.base_variants[0]` (~9883)로 첫 변이만 form이 됨.
- **변이형별 "따로 생기는 전용 스킬"** (`variantSkillsOf`, ~5171, 상단 6칸과 별개인 하단 변이 바):
  - 포식 → `skill_card_predation` (1)
  - 무기 → 장착 무기 스킬 (최대 2)
  - 독성 → `skill_card_infuse`, `skill_card_spray` (2)
  - 용족 → `skill_card_scale`, `skill_card_breath` (2)
  - **포자 → 없음 (0)** ← 6변이형 중 유일하게 전용 스킬을 안 줌.
- **포자가 실제로 하는 일:** ① 포자 카드(마비/중독/방해) 장착 자격 + ② 장착한 포자 카드가 **매 턴 끝에 확률(30~40%)로** 적에게 상태이상 발동(`tickStatuses` 내 `u.cardFx.spores` 루프, ~12882, `roll < sp.chance`). 포자 카드의 `endTurn` → `cardInstanceEffects`에서 `out.spores`로 변환(~5261).
- **"포자속성 스킬을 강화하는 패시브"는 현재 없음.** (사용자 추측 정정)

### 1. 이중 변이 버섯 = `baseVariants:['spore', X]`
- `baseVariants`는 이미 배열 — 첫 원소(spore)가 form·외형·정체성·컨셉축을 결정. 두 번째 변이는 "카드/스킬 호환성"으로만 표현.
- 외형·스탯은 그대로 **버섯**. (버섯은 저스탯 by design — `applyVariantIdentity`에서 `form==='spore' || seedType==='mushroom'` 제외, ~6265. 이중버섯도 첫 변이 spore라 동일하게 제외 → 버섯 밸런스 유지.)
- 자기검증 `__catalogSelfTest`의 "버섯은 태생 spore" 단언(~14266)은 `baseVariants[0]==='spore'`만 보므로 `['spore', X]`도 통과.

### 2. 융합 전용 메커니즘은 **만들지 않는다** (= "포자 적층" 아이디어 폐기)
브레인 도중 "포자 적층(쌓고→터뜨린다)" 신규 자원을 검토했으나 **폐기.** 이유: 포자는 스킬 칸을 안 먹는 패시브 아우라라서, 두 번째 변이의 스킬/카드를 그대로 얹어도 **스킬 수가 안 터진다.** → 융합 로직을 따로 안 짜도 "포자가 깔린 채 포식기를 쓴다" 같은 융합감이 **공짜로** 나옴. **공존(coexistence)으로 충분.**

### 3. 그래서 필요한 구현 = **두 함수를 `base_variants` 전체로 확장** (병렬, 동일 패턴)
지금 둘 다 `p.form`(단수=spore)만 봐서, 그냥 두면 두 번째 변이의 카드·스킬이 **안 뜬다.**
- **(가) `cardFitsForm`** → 카드가 `common`이거나 **종의 변이 중 하나라도** 일치하면 장착 가능. (dragon이 두 번째 변이여도 spore가 common을 허용하므로 공통 카드 정상 장착 — 모서리 케이스 자동 해소.)
- **(나) `variantSkillsOf` + `makeCombatant`의 born-skill push** → `p.form` 대신 `base_variants` 전체를 순회해 두 번째 변이의 전용 스킬(포식기/주입·분사/비늘·브레스 등)을 부여.
- 기존 변이 카드(독성/포식/무기/용족)는 이미 `TRAIT_CARDS`에 존재 → **시스템 자체엔 새 카드 0개.**

### 4. 포자 변이 재정의 (독립 트랙 — 기존 순수 포자 버섯 7종에도 적용)
사용자 의도: **"매 턴 끝 포자 발산 이벤트는 항상 발생(보장)"** 하되, 그 발산에 올라탄 **각 효과의 적중은 효과별 확률로 별개 판정.**
- 현재도 `tickStatuses`가 매턴 spores를 굴리지만, 사용자 모델은 "발산은 항상 + 효과별 확률"로 프레이밍. 효과별 확률을 유지하므로 **"영구 마비" 같은 밸런스 폭주 없음**(밸런스 자동 해소).
- 이 트랙은 이중버섯과 **독립** — 포자 시스템 전반 개선이라 순수 포자 버섯에도 반영.

## 미정 (다음에 다시 고민할 것)

1. **포자 발산에 "올라타는 효과"의 범위** — 미확정. 두 안:
   - A) **포자 카드 효과만**(마비/중독/방해): 지금과 같은 대상, "항상 발산 + 효과별 확률"로 다듬기만. 단순.
   - B) **장착된 변이 카드 효과 전부**(포식/무기/독성 카드도 매턴 발산 = 운반체 융합).
   - ⚠️ **B의 난점(중요):** 비-포자 변이 카드는 효과 형태가 다름 — 무기 카드는 *스킬 부여*(grantSkill), 포식 카드는 *흡혈 패시브 배율*(predBoost), 독성 카드는 *중독 증폭*(poisonCoef). "매 턴 끝 적에게 상태 발동"으로 깔끔히 매핑되는 건 포자 카드뿐. → B를 하려면 각 변이가 "발산용 효과"를 무엇으로 기여할지 별도 정의 필요. (사실상 #2에서 폐기한 융합 설계의 축소판이 재등장) → 현재 흐름상 **A가 자연스럽고, 융합은 #3 공존으로 해결**되는 쪽이 정합적. 재개 시 최종 결정.
2. **포자 "항상 발산"의 밸런스 수치** — 효과별 확률 유지가 전제. 정확한 확률/수치 패스 TBD.
3. **두 번째 변이 종류 선택** — toxic/pred/weapon/dragon 중 어디까지. (추천: 4종 전부 = 변이당 1마리)
4. **마릿수** — 변이당 1마리 vs 속성별로 촘촘히. (현재 순수 포자 버섯이 이미 7속성 전부 커버 → 새 축은 "두 번째 변이"가 자연스러움.) 추천: **4종(변이 종류당 1마리)**.
5. **개체별 단계 스킬·고유스킬 부여** (생성기 `scripts/gen-*.js` 경로), **탐사 분포**(`MUTANT_SIGNATURES` + EXPLORE_VIEW), **released 플래그** — 전부 후속.
6. 결정론 규율: 신규/수정 전투 코드(포자 발산·variantSkillsOf 확장)는 `rng()` 단일 함수·순수 함수 규율 준수.
