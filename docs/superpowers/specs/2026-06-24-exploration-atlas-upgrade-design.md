## 0. 한 줄 요약
행성·지역에 **특정 종이 등장**하도록(포켓몬 지역 감성) 종 분포를 바꾸고, "왜 주변 행성마다 생명이 있나"를 **아틀라스 세계관**(여러 외계 문명이 공유한, 다른 은하의 발견된 좌표들 + 시공간 폴드 이동)으로 해소한다. 경기장(풀로세움)은 **항상 열린 게이트**라 무료 입장으로 정리한다. **속성 상성표·타입 체계는 일절 건드리지 않는다.**

## 1. 배경 / 문제
- 현재 탐사는 지역에 `el`(속성)·`types`(타입) 배열만 있고, `rollSpeciesFromView`가 그 조건에 맞는 종 풀에서 희귀도 가중으로 뽑는다 → *조건 필터*일 뿐 "이 행성엔 이 개체가 산다"는 **지역 정체성**이 약하다. 종을 대거 추가할 예정이라 지역색이 더 중요해졌다.
- 우주맵의 8행성이 전부 식물 서식지 → "이웃 행성마다 다 생명이 산다"는 **비과학적 인상**.
- 탐사는 연료를 쓰는데 전투(경기장 입장)는 아무 비용이 없어 **경제 비대칭**이 어색하다.

## 2. 확정 결정 (브레인스토밍 결과)
1. **종 분포 = 행성 풀 + 지역 시그니처 (옵션 C).** 행성에 "서식 종 풀"을 두고, 지역은 기존 `el`/`types`로 그 풀을 좁힌다. 지역마다 "여기서만, 낮은 확률로" 나오는 **희귀 시그니처 종 1~2개**를 콕 지정한다.
2. **세계관 = 아틀라스.** 행성들은 한 태양계 이웃이 아니라, **여러 외계 문명이 각자 다른 은하에서 탐사해 공유한 "발견된 세계들의 목록"**. 플레이어는 **시공간을 접는 초월 기술(폴드 점프)** 로 해당 좌표로 이동한다. → 목록에 오른 곳은 애초에 생명·시료가 있는 세계라서 "왜 다 생명이 있나"가 자연 해소.
3. **지도 처리 = 로어 재해석(A) + 다른 은하 분위기 + 차원이동 모션.** 궤도 맵 **구조는 그대로**, 의미만 교체 + 배경 은하 앰비언스 + 탐사 실행 시 폴드(차원 이동) 연출.
4. **연료 = 폴드 에너지.** 로켓 연료가 아니라 "공간을 접는 막대한 에너지". 멀고 깊은 좌표일수록 더 듦. 기존 "연료탱크 레벨 → 바깥 궤도 해금"이 "폴드 출력↑ → 더 먼 심도 해금"으로 그대로 맞아떨어진다(명칭/설명만 리스킨).
5. **경기장 = 항상 열린 게이트, 무료 입장.** 풀로세움은 모든 문명이 항구적으로 열어둔 고정 폴드 게이트라 접는 비용이 0 → 연료·참가권 없음. (로드맵 §7 "참가 비용 둘지" → **안 둠**으로 확정.) 안내 문구만 이 로어로 보강.
6. **속성 상성표·타입 5종 체계는 변경 없음.**

## 3. 데이터 모델
`EXPLORE_VIEW`(index.html, 약 6495줄~)에 필드 추가. 기존 필드 무변경, 신규는 옵셔널(없으면 폴백).

```js
// 행성 레벨 — 신규 species 풀
{ id:'ex_calisto', name:'칼리스토', orbit:1, …,
  species:['ember_cactus','dew_cactus','clayblum', …],   // ← 신규: 이 행성 서식 종(key 목록)
  regions:[
    { name:'세이라 분지', el:['earth','fire'], types:['다육형'],  // 기존 필터 유지
      signature:['rare_xxx'],   // ← 신규(옵셔널): 이 지역에서만, 낮은 확률
      diff:3, grade:'C', cost:200, … }
  ]}
```

- `species`: 비-legacy 종 key 배열. 없으면 기존 동작(el/types 전역)으로 폴백.
- `signature`: 지역 전용 희귀 종 key 1~2개. 없으면 시그니처 단계 생략.

## 4. roll 로직 (`rollSpeciesFromView` 개정)
입력: 지역(viewRegion) + 그 지역이 속한 행성(planet). (호출부에서 planet도 넘기도록 시그니처 보강.)

1. **시그니처 판정:** `region.signature`가 있으면 `SIGNATURE_CHANCE`(초기값 0.10) 확률로 그 중 1종을 반환.
2. **일반 풀:** `pickAcquirableSpecies(filter)` 재사용, filter =
   - `planet.species`가 있으면 그 풀 안에서 + 지역 `el`/`types` 일치
   - `planet.species`가 없으면 기존처럼 `el`/`types` 전역
3. **폴백 사다리(무회귀):** (행성 풀 ∩ 지역 필터)가 비면 → 행성 풀 전체 → 그래도 비면 → el/types 전역(기존 동작). 어떤 데이터 상태에서도 항상 1종을 반환.

`SIGNATURE_CHANCE`는 상수로 빼서 조율 가능하게 한다.

## 5. UI (복잡도 억제 — 사용자 핵심 제약)
- **행성 팝업(그림2):** "주요 서식종" **아이콘 3~4개**만 미리보기. 전체 종 나열 금지. (행성 `species` 풀에서 대표 추출 — 예: 희귀도 높은 순 또는 앞쪽 N개.) `species` 없으면 이 줄 생략.
- **지역 상세(그림3):** 지역을 선택했을 때만, 그 지역에서 만날 수 있는 종을 **작은 아이콘 줄**로. 시그니처 종은 `✦희귀` 뱃지로 구분. 공간 부족하면 가로 스크롤 한 줄.
- 아이콘은 기존 절차적 SVG(`composePlantSvg`/`spriteFor`) 재사용 — 새 에셋 불필요.

## 6. 세계관 리스킨 (지도 A + 모션)
- **명칭/문구 교체(코드 영향 최소):** 중앙 항성 → "폴드 게이트(모함)", 궤도 링 → "폴드 심도 1/2/3", 행성 카드 → "다른 은하의 발견된 좌표". 연료 → "폴드 에너지"로 라벨/툴팁/설명 리스킨. 행성 `intro`/`themeNote`를 아틀라스 로어("여러 문명이 공유한 좌표")로 갱신.
- **배경 앰비언스:** 우주맵 배경에 은하/성운 느낌(절차적 SVG/그라디언트/별 점). 가벼운 CSS/SVG로.
- **차원 이동 모션:** 탐사 실행 시 화면 왜곡/수축 → 파동 링 → 점선 폴드 경로 → 좌표 도착 연출. `prefers-reduced-motion`이면 짧은 페이드로 축약.
- **경기장 안내 문구:** 함선 콘솔/토너먼트 진입부에 "항상 열린 게이트라 폴드 비용 없음" 취지의 한 줄 로어.

## 7. 회귀 안전 / 테스트
- 신규 필드는 전부 옵셔널 → `EXPLORE_VIEW`를 한 번에 다 채우지 않아도 동작. 행성 풀은 점진적으로 채운다.
- `window.__test(...)`로 셀프테스트 추가:
  - `rollSpeciesFromView`가 **행성 풀 안에서만** 뽑는지(풀 지정 시).
  - 지역 `el`/`types` 필터가 **여전히** 적용되는지.
  - 시그니처가 **일반 풀에 섞여 나오지 않고** 별도 확률 경로로만 나오는지(확률 0/1 강제로 검증).
  - (행성 풀 ∩ 지역 필터) **빈 교집합일 때 폴백**이 항상 1종 반환하는지.
- preview로 우주맵→행성 팝업→지역 상세→탐사 연출 시각 확인.

## 8. 범위 밖 (후속 작업)
- **행성 종류 확장**(사용자 요청 "이후 행성도 늘리자"): 본 구조가 깔린 뒤, 새 행성/지역 + 종 풀/시그니처를 데이터로 추가. 별도 후속 작업으로 진행.
- 종 자체의 대량 추가(#1)도 별도. 본 작업은 **분포 구조 + 세계관 + UI**까지.
- 탐사 거리/은하 확장(해금형)은 폴드 심도 리스킨으로 토대만 마련, 본격 확장은 후속.

## 10. 점검에서 나온 후속 개선점 (2026-06-24 감사 — **나중에 작업**)
