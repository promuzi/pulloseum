# 개체 컨셉 정립 + 단계별 스킬 다양성 대규모 확장 — 설계

> 2026-06-26 · #1-A "변이 개체 정체성"의 후속. 1차(아키타입 축: 스탯·이름·DoT)는 완료. 이번은 **컨셉 먼저 → 그 컨셉에 맞는 스킬을 단계별로 대규모 추가**.

## 배경 / 문제
1차로 개체마다 아키타입(성격) 14종이 붙어 **이름·스탯·DoT 종류**는 달라졌다. 하지만:
- **싸우는 방식은 아직 거의 같다** — 효과/구조는 1차에서 의도적으로 안 건드림(밸런스 보존).
- 개체 스킬은 단계별로 `ind.<key>.g/m/e` **1개씩**뿐. 새싹·유체는 전 개체 공유(`STAGE_SKILLS`). → 같은 타입+속성이면 스킬 구성이 거의 동일.
- 결과: 컨셉이 흐릿하고, 개체 간 플레이 차별화가 약하다.

## 목표 (사용자 확정 방향)
1. **컨셉 우선:** 먼저 각 개체의 컨셉·개성·간단한 스토리를 정하고, **그 컨셉에 어울리는 스킬을 골라 넣는다.** (스킬 효과부터 정하지 않는다.)
2. **유동적 2축 블렌딩:** 개체 정체성은 하나의 중심이 아니라 **속성+변이 / 속성+타입 / 타입+변이** 같은 **2축 묶음**으로 또렷하게 한다. (단일 중심 강제 금지)
   - 예: "포식형이지만 불 정령 식물" → 속성(불)을 우선축으로 잡아야 개성이 산다. 포식은 보조축.
3. **단계별로 스킬을 적극 추가** — 새싹(간단 손질) / 유체 +1 / 성장체 +2 / 성체 +2 / 완숙체 캡스톤.
4. **단순 스펙업 금지** — 새 스킬은 전부 **역할·비용으로 차별화**해서, 옛 스킬도 (싸거나 상황상) 계속 쓸모 있게.
5. **모두가 변이 시그니처를 갖지 않는다** — 컨셉이 결정. 같은 포식이라도 어떤 개체는 새싹에 포식 스킬, 다른 개체는 속성 스킬.
6. **대규모 다양성** — 약 28 비버섯 base + 140 변이(+ 버섯 7) 개체에 걸쳐.

## 핵심 개념: 컨셉 = 2축 블렌드 + 성격 + 스토리

```
개체 컨셉 = (주축 ⊕ 보조축)  +  아키타입(효과 경향)  +  스토리 한 줄
            └ 속성/변이형/타입 중 둘 ┘
```
- **주축·보조축** = `속성 / 변이형(form) / 타입` 중 둘을 묶음. 어느 쪽이 주(主)냐로 스킬 색깔의 비중이 갈린다(주축 스킬을 더 많이/이른 단계에).
- **아키타입** = 1차에서 정한 성격(광폭·도사·…). 스킬의 **효과 경향**을 부여(아래 §효과 경향 표). 컨셉의 "성격" 레이어.
- **스토리** = 1~2문장 짧은 설정. 도감 노출(기존 `SPECIES.desc` 경로 재사용 → 컨셉문 + 스토리).

### 데이터 구조 (편집 표면 = 아키타입과 동일 철학)
- `CONCEPT_OVERRIDES = { speciesKey: { primary, secondary, theme, story } }` — **사용자 큐레이션 표면.** 비면 자동 시드.
- `conceptOf(key)` = `CONCEPT_OVERRIDES[key] || autoConcept(key)`.
  - `autoConcept`: (타입·속성·form·아키타입) + 안정 해시로 **주/보조축을 다양하게 분산**. 변이형 개체는 form을 축 후보에 포함, base는 타입·속성 중심. 나중에 override로 손맛.
- 모든 개성은 `CONCEPT_OVERRIDES` + `ARCHETYPE_OVERRIDES` 두 표에 모임 → 한 곳에서 큐레이션.

## 단계별 스킬 예산 (개체당)
| 단계 | 현재 | 목표 | 비고 |
|---|---|---|---|
| 씨앗 | 0 | 0 | — |
| 새싹(sprout) | 공유 1~2 | 공유 + **간단 1**(컨셉 가벼운 맛보기) | 저비용 견제/기본 색깔 |
| 유체(juvenile) | 공유 | 공유 + **개체 1** | 주축 색깔 1개 |
| 성장체(growing) | 개체 1 | 개체 **2** | 주축 + 보조축/성격 |
| 성체(mature) | 개체 1 | 개체 **2** | 역할 분화(공격/유틸) |
| 완숙체(evolved) | 개체 1 | 개체 **1 캡스톤** | 그 개체의 최고 시그니처 |
- 부착 위치 = `SPECIES_CATALOG[key].stageSkills[stage]` 배열에 push, 스킬 정의는 `SKILL_LIB`.
- **모든 개체가 같은 수를 강제로 받지 않는다** — 컨셉이 빈약하면 적게, 진한 개체는 꽉. 위는 상한 가이드.

## 효과 경향 표 (아키타입 14종 — 새 스킬이 이 경향을 따른다)
> 변이형과 직접 겹치는 효과는 피한다(흡혈=포식, 관통=무기, 중독=독성). 정말 특수할 때만.

| 성격 | 경향 |
|---|---|
| 광폭(berserk) | **출혈** 위주 공격 |
| 냉혹(executioner) | **단일 큰 피해 + 출혈** |
| 야성(feral) | **출혈 + 자기 기동력↑**(연속 사냥) |
| 도사(sage) | **단일 큰 피해 + 치명타 높음** |
| 교활(trickster) | **적 기동성↓ + 치명타** |
| 망령(phantom) | **적 적중↓ + 자기 기동↑**(환영) |
| 수호(guardian) | **방어막 + 반격**(공격 약) |
| 우직(stalwart) | **반격 + 자기 방어↑** |
| 책략(strategist) | **자기 버프 + 상대 디버프** 위주 |
| 폭군(tyrant) | **적 공격력↓(위압) + 위력↑** |
| 맹장(warlord) | **위력↑ + 진격(자기 공격↑)** |
| 은둔(hermit) | **회피(기동성)↑** 위주 |
| 광신(zealot) | **자해 화상 + 자기 공/방↑**, 화상·출혈 보너스 |
| 탐식(glutton) | **에너지 흡수** |

### 효과 경향과 균형(아키타입별 트레이드오프)
- 공격형 성격(광폭·도사·냉혹·야성·맹장·폭군) = 순증 허용.
- 방어/유틸형(수호·우직·은둔·책략) = 위력 살짝↓ + 유틸↑.
- 14종을 완전 직교(겹침 0)는 엔진 노브 한계로 불가 → **같은 원자를 다르게 조합**해 구별(예: 출혈 가족 광폭/냉혹/야성을 단일대미지·자기기동 조합으로 분리).

## ⭐ 확장 원칙 (가장 중요)
> 이 문서의 **모든 메커니즘 표(효과 경향 14·form 시그니처·엔진)는 "씨앗 예시"**일 뿐이다. 구현 시 각 축마다 **여러 변형으로 다양하게 늘린다.**
- form 하나당 시그니처 스킬 **여러 종**(예: 무기형 = 속성 부여 / 일점 관통 / 무기 변형 / 연격 …), 성격 하나당 여러 변형, 속성·타입도 마찬가지.
- 같은 form·성격이라도 개체마다 **다른 변형**을 받아 겹침을 줄인다(컨셉 블렌드가 어느 변형을 줄지 결정).
- 목표는 "표대로 14/5개"가 아니라 **풍성한 스킬 패밀리**. 생성기 + 큐레이션으로 폭을 키운다. 표는 방향만 고정.

## 변이형(form) 시그니처 메커니즘 → 개체 스킬
> 컨셉 블렌드에 **변이형 축**이 들어간 개체는 그 form의 시그니처 메커니즘을 스킬로 갖는다(개체마다 조금씩). 효과 경향(성격)과 별개의 "form 색깔" 레이어. **아래는 씨앗 예시 — 각 form을 여러 변형으로 확장한다(위 확장 원칙).**

| form | 시그니처 컨셉 | 엔진 |
|---|---|---|
| 무기(weapon) | **속성 부여** — N턴간 공격에 자기 속성 피해를 얹음("무기에 속성을 입힘") | ⚠️ 기존 `infuse`(독 전용)를 일반화 → `infuse:{kind:'element'}` 또는 신규 `elemInfuse`. **신규(소)** |
| 독성(toxic) | **독 강화** — 기존 `infuse`(독 묻히기)·`spray`(독 뿌리기) + 자기 독 위력↑ 버프 | ⚠️ 기존 재사용 + 자기 독 증폭 버프 1종 **신규(소)** |
| 용족(dragon) | **브레스 준비** — 충전 스킬로 스탯/스택 쌓고, 브레스가 스택에 비례해 강화 | ✅ 미사용 `rampStack` 필드 활용(충전형 atk 누적) + breath power 스케일. 비늘(`scaleStack`=방어)과 별개 |
| 포자(spore) | **포자 살포/잠복** — 매턴 포자(기존 `spores`) + 누적 확산 디버프 | ✅ 기존 `spores`/`infuse` 계열 재사용 |
| 포식(pred) | **흡혈·강탈**(기존 `pred`) — 포식 전용 | ✅ 기존. 흡혈은 포식 정체성이라 **다른 성격엔 안 줌**(중복 회피 원칙) |

- form 시그니처는 **자동 부여 base form 스킬**(`skill_card_*`)과 **별개의 개체 스킬**로, 컨셉이 진한 개체에 단계별로 1~2개 섞는다.
- 무기형 "속성 부여"는 컨셉이 속성 주축일 때 특히 강조(사용자 예: 불 정령 무기형 → 속성 부여가 핵심).

## 엔진 변경 (작음 — 새 시스템 + 경향 적용부)
1. **출혈(bleed) 전역 = 지속피해 + 회복 감소.** `bleed` 상태가 활성인 동안 그 대상의 회복(`s.heal`·`lifesteal`·`regenPct`)에 배수(예: ×0.5) 적용. 적용 지점 = `applySkill` 회복부(line ~11616), 흡혈부(~11693), `tickStatuses` 재생부(~11720). 헬퍼 `healMult(u)` 신설. **전역**(누가 걸든 동일) — 출혈의 정체성.
2. **에너지 흡수(탐식)** — 스킬 필드 `energyDrain:n` 신설. 피해 적용 후 `d.energy → a.energy` 이전(기존 `pred` 카드의 `energySteal` 로직 재사용, line ~11697). 봇·플레이어 공통.
3. **회피(은둔/망령)** = 신규 메커니즘 불필요. 회피는 이미 `30 + (방어자 기동성 − 공격자 적중)` 구조(line ~11632). 은둔=자기 기동↑(selfBuff spd), 망령=적 적중↓(enemyDebuff acc)로 표현.
4. **광신 자해 화상 + 공/방 동시↑** — 엔진은 스킬당 selfBuff 1개 제한 → 광신 시그니처는 **전용 처리**(자기 dot:burn + 버프 2개 수동 push)로 소수 bespoke 스킬만. 일반 경로 아님.
5. **아키타입 효과 경향 적용** — 1차 `applyVariantIdentity` IIFE(line ~5322)와 별개로, 생성기가 새 스킬을 만들 때 경향 표를 참조해 효과를 부여(데이터에 박힘). 런타임 IIFE는 1차 그대로(이름/DoT/스탯) 유지.
6. **무기형 속성 부여** — 기존 poison 전용 `infuse`(line ~11623, `poison_infuse` 버프)를 일반화. `infuse:{kind:'element'}`면 `element_infuse` 버프 → 공격 적중 시(line ~11703 동일 위치) 자기 속성 추가 피해. **신규(소).**
7. **독성 자기 독 증폭** — 자기 buff `poison_amp`(독 위력 ×N, M턴) 1종. 적용 시 `infuse`/`dot poison`/`spray` 데미지에 곱. **신규(소).**
8. **용족 브레스 충전** — 미사용 `rampStack`(line ~11088 초기화 존재) 활용: 충전 스킬 `ramp:{atkPct,cap}`이 `rampStack`++ & atk 누적 버프, 브레스 스킬 `breathScale:true`면 power가 `rampStack`에 비례. **신규(소, 필드 추가).**

## 빌드 방식
- **생성기 확장:** `scripts/gen-individuals.js`에 컨셉 레이어 + 단계별 스킬 생성을 추가(멱등·재실행 안전). 한국어 컨셉 서술 → 엔진 필드 기계 변환(기존 매핑 규칙 계승).
- **컨셉 데이터:** `CONCEPT_OVERRIDES`(큐레이션) + `autoConcept`(자동 시드). 스토리/컨셉문은 도감 자동 노출.
- **단계별 진행:** 개체 전체를 한 번에 X. **단계(또는 개체 배치)별로 나눠** 생성→검증→커밋. 각 배치는 컨셉+스토리+해당 단계 스킬이 완결된 단위.

- **P1~ — 단계별/배치별 콘텐츠:** 컨셉+스토리 정립 → 단계별 스킬 생성(역할·비용 차별화). 배치마다 검증·커밋·도감 확인. 사용자가 배치 결과를 보고 큐레이션.

## 비고
- 개성은 두 표(`CONCEPT_OVERRIDES` + `ARCHETYPE_OVERRIDES`)에 수렴 → 사용자가 한 곳에서 개체별 손맛.
- "단순 스펙업 금지"는 생성기 규칙으로 강제: 새 스킬은 반드시 역할 태그(견제/대박/셋업/유지/회피 등) + 비용 스펙트럼을 갖고, 순수 stat-only 스킬은 거부.
