# 속성별 시그니처 성질(상태이상) — 설계서

> 2026-06-26 박제. 로드맵 [§4-14](../../master-roadmap.md#14-속성별-고유-성질상태이상--속성-개성화)의 확정 설계.
> 목표: 지금 속성은 **상성 배율(약점 1.5×/저항 0.67×)만** 차이가 난다. 여기에 **속성마다 고유한 "성질"** 을 얹어, 같은 위력이라도 속성에 따라 전투 맛이 달라지게 한다. **속성 = 정체성.**

## 1. 부여 범위 (확정)

- **속성기(`ELEMENT_SKILLS`, `skill_elem_*`) + 속성발현(속성 단계 스킬)에만 자동 부여.**
- 일반/변이/개체 스킬에는 붙지 않는다. (속성 전용 스킬을 쓸 때만 그 속성의 성질이 발동)
- 데이터 주도: 상성표(`ELEMENTS`)에 성질 키를 1개씩 추가(`ELEMENTS.fire.signature='burn'` 등)하고, 속성 스킬 생성 루프(`ELEMENT_SKILL_FX` → `ELEMENT_SKILLS`)에서 성질을 스킬에 실어 보낸다.
- ⚠️ 기존 속성기 라이더(물=자힐·대지=방어무시·바람=자가속·번개=치명타 등)는 **아래 시그니처로 교체/조정**한다. 불=화상·풀=흡혈 계열은 사실상 유지(풀은 흡혈→고정 자가회복으로 변경).

## 2. 7속성 시그니처 성질 (확정)

| 속성 | 성질 | 효과 | 대상 | 판정 |
|---|---|---|---|---|
| 🔥 불 | **화상** | 매 턴 지속 피해(DoT) | 적 | 확정 |
| 💧 물 | **젖음** | 기동성↓ + 방어력↓ | 적 | 확정 |
| 🌿 풀 | **재생** | 적중 시 소량 **고정** 자가회복(자기 최대체력 비례, **피해량과 무관**, 흡혈보다 낮게) | 자기 | 확정 |
| ⚡ 번개 | **감전** | 적중 시 **확률** 즉시 스턴 + 감전 스턴 턴에는 에너지 회복 차단 | 적 | 확률 |
| 🪨 대지 | **관통** | 적 방어 일부 무시. **상대가 방어 태세거나 방어 버프 중이면 관통 효율↑** | 적 | 확정 |
| ❄️ 빙결 | **빙결** | 적중마다 빙결 스택(2턴 유지). 스택 수 비례 **기동성↓ + 적중률↓**. **3스택 도달 시 확정 스턴 → 스택 0 초기화** | 적 | 결정론(스택) |
| 🌪️ 바람 | **연속타** | 주 타격 후 **약한 추가타**. **기동성 비례로 추가타 횟수 증가**("파바밧→팟" 연출) | 자기 | 확정 |

### 속성별 상세

- **🔥 불 / 화상:** 기존 `addDot('burn')` DoT 재사용. 매 턴 시전자 공격력 기반 소량 피해.
- **💧 물 / 젖음:** 적에게 spd↓ + def↓ 이중 디버프(시그니처 전용 상태, 스킬의 `enemyDebuff` 1개 제약과 별개로 엔진이 직접 부여). "젖어서 둔하고 무름."
- **🌿 풀 / 재생:** 흡혈(피해 비례)이 아니라 **적중 시 자기 최대체력의 고정 비율 회복**. 흡혈보다 수치를 낮게. 저위력 스킬로도 꾸준히 버티는 결.
- **⚡ 번개 / 감전:** 적중 시 일정 확률로 즉시 스턴(다음 자기 행동 1회 스킵). **스택·턴 누적 없음 — 순수 RNG.** 감전 스턴 중인 턴에는 그 유닛의 턴당 에너지 회복이 막힌다(행동자원까지 정지).
- **🪨 대지 / 관통:** 적 방어력의 일부를 무시(전체 무시 아님). 대상이 그 턴 방어(`guard`) 태세이거나 방어 버프(def selfBuff) 보유 시 **관통 비율 가중**. "단단함을 뚫는다."
- **❄️ 빙결 / 빙결:** 적중마다 스택 +1, 각 스택 **2턴 유지**(추가 적중으로 갱신, 안 맞으면 만료). 스택 수에 비례해 spd↓ + acc↓ 약하게. **3스택 도달 즉시 확정 스턴**, 발동 후 스택 0 초기화.
- **🌪️ 바람 / 연속타:** 스킬의 주 타격 직후 약한 추가타 N회. **N = 시전자 기동성에 비례**(기준값 + spd 비례 가산). VFX: 빠른 잔타 여러 번 → 마무리 한 방.

## 3. 공통 규칙 (확정)

- **상성 연동 없음** — 성질은 상대가 그 속성의 약점인지와 **무관하게 항상 동일 강도**로 적용.
- **자기 속성 저항(경감, 면역 아님)** — 유닛은 **자기 속성과 같은 성질에만** 효과가 경감된다.
  - 불 식물 = 화상 DoT 경감, 번개 식물 = 스턴 확률 경감, 빙결 식물 = 빙결 스택 덜 쌓임(또는 스택당 디버프 경감), 물 식물 = spd/def 디버프 경감, 대지 식물 = 관통 경감.
  - 풀(재생)·바람(연속타)은 **자기지향**이라 저항 개념 해당 없음.
- **스턴 연속 면역 없음** — 스턴 직후에도 재스턴 가능(번개·빙결 둘 다). 빙결은 어차피 3스택을 다시 모아야 하고, 번개는 운에 맡긴다.

## 4. UI 노출 (확정)

- 스킬 카드 옆 **속성 칩 옆에 성질 아이콘**, 카드 상세에 **성질 한 줄** 설명.
- **도감 자동 반영** — `composePlantSvg`/`__DEX_API` 경로처럼, index.html의 속성 스킬 정의만 고치면 도감(`plant-codex.html`)이 성질 표기까지 자동으로 가져가도록.

## 5. 신규 엔진 요소

새로 필요한 메커니즘:

1. **스턴(행동 스킵):** 유닛에 `stunned` 카운터(턴 수). 턴 진행 루프에서 스턴 상태면 그 유닛의 행동 1회 스킵. 번개 변형: 스턴 턴 동안 에너지 회복(턴당 `energyGain`/회복) 차단.
2. **빙결 스택 카운터:** 유닛에 `freezeStacks`(값·만료 턴). 적중마다 +1/갱신, 2턴 만료, 3스택 → 스턴 + 리셋. 스택 비례 spd/acc 디버프는 스탯 계산 시 반영.
3. **바람 추가타:** `applySkill`의 데미지 적용을 N회 반복(주 1 + 추가 N), N = spd 비례. 각 추가타는 약한 계수. VFX 훅.
4. **대지 관통(가중):** 데미지 계산의 방어 적용부에서, 대상의 방어 태세/버프 여부에 따라 pierce 비율을 가중.
5. **풀 적중 시 고정 자가회복:** 적중 성공 시 시전자 maxHp × 고정 비율 회복(피해 무관).
6. **물 이중 스탯 디버프:** 시그니처가 spd+def 두 스탯을 동시에 깎는 디버프 부여(기존 단일 `enemyDebuff` 제약 우회 — 시그니처 전용 경로).

엔진 재사용: DoT는 `addDot`, 디버프는 기존 스탯 디버프 구조, `DOT_STACK_CAP=4` 등 관례 유지.

## 6. 수치 (밸런스 패스에서 확정 — 현재 TBD)

확률·디버프 %·DoT 계수·추가타 횟수/계수·관통 비율·자가회복 비율·스턴 지속·자기속성 저항 경감률은 **구상 단계라 미정**. 1차 구현은 기존 관례값(화상/출혈 0.06, 중독 0.05, 디버프 0.2~0.3, pierce 0.4 등)을 임시로 쓰고, 별도 밸런스 패스에서 조정한다.

## 7. 구현 순서(제안)

1. `ELEMENTS`에 `signature` 키 + 속성 스킬 생성 루프 연결(데이터)
2. 스턴/빙결 스택 상태 필드 + 턴 루프 분기(엔진 신규)
3. `applySkill`에 속성 시그니처 적용 분기(불/물/풀/번개/대지/빙결/바람)
4. 자기 속성 저항 경감 분기
5. UI: 카드 상세/칩 성질 표기 + 도감 반영
6. 셀프테스트(`window.__test`)로 회귀 검증
