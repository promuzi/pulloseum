## 1. 전투 하단 스킬바 지연 제거 (버그)

**현상:** 마지막 후공 처리가 끝난 뒤 다음 턴 스킬바가 다시 활성화되기까지 약 1초 가까이 걸린다.

**원인:** `playerSkill` 말미 ([index.html:11717~11727](../../../index.html))
```
tickStatuses(p) → tickStatuses(e) → sleep(620) → closeJudge() → busy=false; updateBars()
→ if(!B.over){ sleep(380); renderBattleIdleView(); }
```
판정 닫힘(620ms) + 유휴뷰 복귀 지연(380ms) = 마지막 행동 해소 후 ~1s 동안 스킬바가 잠긴 채 비활성.

**수정 방향:**
- 마지막 행동 해소 직후 **스킬바를 먼저 풀어준다.** `closeJudge()` 직후 `B.busy=false`로 풀고 `refreshSkillBar()`(또는 `renderBattleIdleView()`)를 호출, 이어지는 연출 sleep과 분리.
- 판정 닫힘 대기 `sleep(620)` → 약 **300ms**, 유휴뷰 복귀 전 `sleep(380)` 제거(또는 ~120ms).
- 목표: 마지막 행동이 화면에서 해소된 시점부터 스킬바 재활성까지 **300~400ms 이내.**
- 회귀 주의: `B.busy` 플래그가 풀리는 시점을 앞당기되, 연출(팝업/spriteFx) 완료 전에 다음 `playerSkill`이 시작돼 상태가 겹치지 않도록 가드 확인.

**검증:** preview에서 `playerSkill` 호출 후 `#cardPhase`의 `locked` 클래스가 풀리는 시각을 폴링 측정.

## 2. 흡혈(포식) = 변이형 전용 메커니즘 (밸런스)

**현상:** 일반형 개체의 성체/완숙체 시그니처 스킬 다수(`ind.*`)가 "○○ 포식" 이름 + `kind:'drain'`/`lifesteal`을 가진다. 포식형 식물이 아닌데도 흡혈 스킬을 쓴다.

**규모:** `kind:'drain'` 스킬 102개, "포식" 이름 41개, "흡혈/흡식/흡수" 31개.

**확정 규칙 (사용자: "흡혈 효과 자체를 변이형 전용으로"):**
- **흡혈(`lifesteal>0` / `kind:'drain'`)은 변이형(`p.form !== 'normal'`) 전용 메커니즘이다.**
- **일반형(`form==='normal'`) 개체의 흡혈 스킬은 흡혈이 아닌 "다양한 요소"의 쓸만한 스킬로 교체한다 (사용자: "흡혈 대신 쓸만한 다른 스킬로, 다양한 요소의 스킬을 만들어 다양하게 채워"):**
  - `kind:'drain'` + `lifesteal` 제거. 단순 일반 공격으로 획일화하지 **않는다.**
  - 대신 개체의 **속성·타입·생장단계**에 맞춰 **다채로운 효과**를 부여: 화상/출혈/중독(DoT), 적 디버프(공/방/속 ↓), 자기 버프(공/방/속 ↑), 치명타율↑, 광역(aoe), 관통(pierce), 에너지 회복 등.
  - 엔진 제약 준수: 스킬당 `selfBuff`/`enemyDebuff` 각 1개, 관통 `pierce:0.5`, 화상/출혈 `0.06`, 중독 `0.05`, DoT 스택 상한 4.
  - **위력은 흡혈 제거분을 상쇄해 일반 공격 기준으로 정규화**(흡혈 없는 동급 스킬과 균형). 효과가 강하면 위력 약간 낮춤, 순수 공격이면 약간 높임.
  - 이름의 "포식/흡혈/흡식/흡반"은 효과에 맞는 중립 어휘로 치환. "포식" 단어는 포식 변이형 스킬에만 남긴다.
  - **다양성 목표:** 같은 속성 계열이라도 단계별(g/m/e)로 효과 종류가 겹치지 않게 분산. 전체적으로 DoT/디버프/버프/크리/광역이 고르게 분포.
- **변이형(weapon/pred/toxic/dragon/spore) 개체의 흡혈은 유지.** (사용자가 "변이형 전용"이라 명시 → 포식형뿐 아니라 모든 변이형 보존.)

**판별 기준(중요):** `ind.<key>` 스킬이 일반형 종에 속하는지 변이형 종에 속하는지 구분해야 한다.
- 변이 140종은 `baseVariants:[form]`로 태생 변이 고정(`SPECIES_CATALOG`). 그 종의 `ind.*` 스킬 = 변이형 → 흡혈 유지.
- base 35종 + grid 종 = 일반형 → 흡혈 제거 대상.
- 구현 시 종↔스킬 매핑을 코드로 산출(어떤 종이 어떤 `ind.*`를 쓰는지)해 일반형 소속 스킬만 변환.

**구현 방식:** 손수정 금지. **감사/생성 스크립트**(`scripts/`)로 멱등 처리.
- `audit-skills.js` 확장 또는 신규 스크립트: 일반형 소속 `kind:'drain'` 스킬 목록 산출 → 변환 규칙 적용.
- 변환 후 `window.__catalogSelfTest()` 통과 확인.

**검증:** ① 일반형 종 하나를 심어 완숙체까지 키운 뒤 로드아웃에 흡혈 스킬이 없음 확인. ② 포식 변이형은 흡혈 유지 확인. ③ 셀프테스트 fails 0.

## 3. 변이카드·스킬 장착 고정 (버그)

**현상:** 변이 카드·스킬을 장착해도 화면을 나갔다 오면 전부 해제된다.

**실제 원인(systematic-debugging으로 확정 — 가설이던 @grade 불일치는 오답):** 3중 TDZ/순서 버그.
1. `let loadedFromLocal`가 `let state = loadState()`보다 뒤 선언 → TDZ.
2. `let state = loadState()`가 state 자기 TDZ 중 실행 → loadState→normalizeState→ensureSkillFields→`ownedTraitCardIds`의 `typeof state`가 TDZ ReferenceError → catch가 삼켜 `defaultState()` 반환 → **2026-06-19 이후 브라우저 세이브가 매 리로드마다 전량 소실**.
3. load 중 `ownedTraitCardIds`가 글로벌 `state.trait_cards`(아직 default=빈)를 읽어 장착 변이카드 필터 탈락.

**수정 방향:**
1. 실제 원인 확정: `ownedTraitCardIds()` 반환 키 형식(등급 포함 여부) vs `equipped_trait_cards` 키 형식을 대조(systematic-debugging — 가설 검증 후 수정).
2. 필터 비교를 **등급 무관 base id 기준** 또는 **동일 키 형식**으로 통일. (같은 card_id 1장 규칙과 충돌 없게.)
3. **스킬 로드아웃(`equipped_skills`)도 동일 점검:** 재진입 시 `defaultLoadout`으로 덮어쓰는 경로([:5533](../../../index.html))가 사용자의 수동 장착을 날리지 않는지 확인. `equipped_skills.length`가 있으면 보존하도록 가드.
4. 변경 후 `saveState()`가 즉시 반영되는지(장착 시점에 저장 호출) 확인.

**검증:** preview에서 카드/스킬 장착 → `ensureSkillFields(p)` 강제 호출(재진입 시뮬) → `equipped_trait_cards`·`equipped_skills`가 유지되는지 반환값 확인.

## 4. 변이카드 가독성 (UI)

**현상:** 변이 카드 목록이 공통/변이 구분 없이 섞여 있고 등급 정렬이 약해 한눈에 안 들어온다.

**확정 요구(사용자):**
- **공통(common) / 변이(해당 변이형 전용) 카드를 섹션으로 분리.**
- **등급별로 같은 가로줄에 배치** — 같은 등급(S/A/B/C/D) 카드가 한 행에 모이도록 그리드 정렬.

**수정 방향:**
- 카드 목록 렌더에 2단 구조: `[공통 카드 섹션]` / `[변이형 전용 카드 섹션]`.
- 각 섹션 내부를 등급 내림차순(S→D) 행으로 그룹핑. 등급 헤더(색상칩: 흰D·초록C·하늘B·보라A·주황S, 기존 `등급 색상` 팔레트 [:5267](../../../index.html) 재사용)와 함께 같은 등급 카드를 한 가로줄에.
- 보유 0장 등급/섹션은 숨기거나 흐리게.

**검증:** preview DOM에서 섹션 구분 노드·등급별 행 그룹 존재 확인.

## 5. 스킬 장착 화면 = 보관함과 통일 + 수치 표기 (UI)

**현상:** 스킬 보관함과 전투 장착 카드의 정보 밀도가 달라 혼란. 전투 카드 앞면은 이름/아이콘/비용만 표시.

**확정 요구(사용자):** 보관함·장착 카드를 **같은 형식**으로 통일하고, 카드에 **수치**를 노출.
- **표기 항목: 비용(⚡) + 위력(power) + 등급(S/A/B/C) + 분류칩(공격/방어/버프/디버프/회복 등).**
- 속성 아이콘은 제외(사용자 선택).

**수정 방향:**
- 스킬 카드 렌더 함수를 보관함/장착 공용 컴포넌트로 통일. 변이카드 가독성(#4)과 동일한 정보 밀도.
- ⚠️ **전투 중 하단 스킬바 카드 앞면은 현 정책 유지**(이름/아이콘/비용만 — CLAUDE.md 전투 UI 규약). 이번 통일은 **관리(강화창) 화면의 스킬 보관/장착 뷰** 대상. 전투 인게임 카드와 혼동 금지.
- 분류칩 색상/라벨은 변이 스킬 `cats` 6종(attack/guard/buff/debuff/heal_hp/heal_energy) 분류 체계 재사용.

**검증:** preview DOM에서 관리 화면 스킬 카드에 비용·위력·등급·분류칩 4요소 노드 확인.

## 6. 탐사 궤도 표시 톤다운 (UI)

**현상:** 탐사 우주맵의 궤도 연료 태그(⛽1/🔒1)가 박스/테두리/배경으로 너무 도드라진다.
※최근 #7에서 궤도 **링**을 복원한 상태인데, 사용자는 **링은 유지하고 글자(연료 태그)만 흐리게** 원함.

**확정 요구(사용자): "링은 유지, 글자만 흐리게."**

**수정 방향:** `.exmap-orbit-tag` ([index.html:1065~1068](../../../index.html))
- 박스 느낌 제거: `border` 없애고 `background`(현 `rgba(6,11,20,.8)`) 제거 또는 거의 투명.
- 글자색 opacity 하향 → 은은한 텍스트만(⛽1 / 🔒1).
- 궤도 **링**(`.exmap-orbit`)은 현 상태(옅은 청록 실선) **그대로 유지**.
- 잠금 태그(🔒)는 식별 가능해야 하니 텍스트는 남기되 톤만 낮춤.

**검증:** preview에서 `.exmap-orbit-tag` computed style — `border`/`background` 제거·opacity 하향 확인. `.exmap-orbit`은 변화 없음 확인.

## 7. 양육 열매 "전부 개봉" (기능)

**현상:** `harvestAllPots()`로 전체 수확은 되지만([index.html:5933](../../../index.html)), 수확된 열매 보상은 `openRewardReveal`로 하나씩 터치해 개봉해야 해 번거롭다.

**확정 요구(사용자):** 여러 열매를 **한 번에 전부 개봉.**

**수정 방향:**
- 다중 보상 개봉 흐름에 **"전부 개봉" 버튼/모드** 추가. 누르면 남은 모든 열매를 순차 애니메이션 생략 또는 일괄 처리로 즉시 개봉하고 결과를 한 화면에 합산 표시.
- 단건 개봉 연출(`openRewardReveal`)은 유지하되, 2개 이상일 때 "전부 개봉" 옵션 노출.
- 합산 결과: 획득 종자/카드 목록을 한 번에 요약.

**검증:** preview에서 화분 여러 개 열매 가득 채운 뒤 전체 수확 → "전부 개봉" → 결과 요약에 모든 보상 포함 확인.
