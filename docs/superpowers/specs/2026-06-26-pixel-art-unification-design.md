# 도트(픽셀) 통일 + 사운드 설계

> 작성: 2026-06-26 · 상태: **1·2단계 구현 완료(2026-06-27, 전 개체 런타임 픽셀화)** · 3단계(UI 토큰화)~6단계(사운드) 대기 · 주제: 게임 전체의 일관된 "미세 픽셀" 비주얼/오디오 정체성
> 브레인스토밍 합의 박제. 구현 계획은 별도 plan 문서로 이어진다.

## 0. 목표 한 줄
식물 개체·효과·버튼·모션·UI·사운드까지 **하나의 "미세 픽셀(분위기용 도트)" 톤으로 규정화**한다. 거친 8비트가 아니라 격자가 은은히 느껴지는 모던/16비트풍.

## 1. 현황(출발점) — 이미 절반 깔려 있음
- ✅ 픽셀 폰트(Galmuri 계열), `image-rendering:pixelated`, 픽셀 그림자 변수(`--px-sh`), 네온 색.
- ✅ 칩튠 사운드 엔진 `sfx`(WebAudio 오실레이터, `beep()` 기반) — tap/hit/heal/roll/win/lose 6종(`index.html` ~10084).
- ⚠️ **핵심 식물·효과는 도트가 아님:** `composePlantSvg`(~10299)는 부드러운 벡터 곡선(path `q`, ellipse)으로 절차 생성 → 픽셀 폰트와 톤 어긋남. 효과(`spriteFx`/`fxPopup`)도 이모지+벡터 혼용.
- ✅ 자산 오버라이드 훅이 이미 존재: `SPRITE_OVERRIDES`·`NAME_OVERRIDES`(~10115), `POT_SPRITE_OVERRIDES`.

**즉 과제 = "절차 생성되는 식물/효과를 어떻게 일관되게 도트화하느냐" + "사운드 다양화하되 톤 유지".**

## 2. 확정된 결정 (브레인스토밍 합의)
| # | 결정 | 내용 |
|---|---|---|
| D1 | 도트 강도 | **미세 픽셀(~96 논리 격자), 모던/16비트풍.** 거친 8비트 아님. |
| D2 | 식물 방식 | AI가 그린 **비픽셀 그림**을 넣고 **엔진이 런타임 픽셀화**. AI 그림 없는 종은 **기존 절차 SVG**를 쓰되 **같은 필터를 통과** → 모든 식물 동일 톤(커버리지 공짜). |
| D3 | UI/효과 | 버튼·HP바·카드·모달·전투 이펙트는 **처음부터 도트 톤**으로 토큰화. |
| D4 | 모션 | AI 프레임 → **가로 스프라이트시트** → **스프라이트 애니메이터(canvas)**, 프레임당 픽셀화 1회 굽고 캐시. |
| D5 | 사운드 | **칩튠 SFX를 전 이벤트 기본 레이어**로 + **AI 오디오/BGM은 임팩트 큰 곳 오버라이드**(없으면 칩튠 폴백). 시각 하이브리드와 동일 철학. |
| D6 | 자산 저장 | **외부 `assets/` 폴더** (data:URI 인라인 아님). |

## 3. 핵심 규정 ⭐ — "전역 픽셀 단위"
일관된 도트가 깨지는 **1순위 원인 = 요소마다 픽셀 크기가 제각각**. 식물은 곱게 픽셀화됐는데 버튼 테두리·그림자·아이콘 도트가 다른 크기면 "도트 게임"이 아니라 "도트 흉내"가 된다.

- **규정:** 화면상 "도트 1칸 = N px"를 **단일 CSS 변수 `--px`(예: 3px)** 로 고정.
- 식물 픽셀화 격자, 버튼 테두리 두께, 픽셀 그림자 오프셋, 아이콘/간격, 폰트 크기 → **전부 `--px`의 배수로만**.
- **제한 팔레트 규율:** 속성별 색 램프(이미 `ELEMENT_PALETTE` 존재)를 기준으로 픽셀화 시 색 수를 줄여(quantize, 예 16~24색) 톤 통일.

이 한 줄 규칙이 "규정화/일관성"의 80%다. 모든 후속 구현은 이 단위를 참조한다.

## 4. 컴포넌트 설계

### 4.1 픽셀화 필터 (식물·이펙트 공용)
- **단일 함수** `pixelize(source, opts) → canvas/dataURL`
  - 입력: HTMLImageElement(AI 그림) 또는 SVG 마크업/엘리먼트(절차 식물).
  - 절차: ① 작은 해상도(~96, `--px` 연동)로 다운샘플 → ② 색 줄이기(quantize) → ③ `imageSmoothingEnabled=false`로 표시 크기까지 확대.
- **단일 진입점 원칙:** `composePlantSvg` 산출물과 AI 오버라이드 그림 **둘 다 이 필터를 거쳐** 화면에 나간다 → 톤 자동 일치.
- **캐시:** 개체 외형 키(종·단계·속성·form·grade…)로 결과 캐시(같은 외형 반복 굽기 방지).
- 경계 조건: `?dex=1`(도감)·preview·`__catalogSelfTest()`와 충돌 없게(필터는 순수 함수, 부팅 비의존).

### 4.2 스프라이트 애니메이터 (모션)
- 입력: 가로 스프라이트시트 PNG + 메타(`{frames, fps, loop}`).
- `<canvas>`에 매 틱 현재 프레임만 잘라 그림. **픽셀화는 프레임당 1회 굽고 캐시** 후 재생(매 프레임 재픽셀화 금지 — 모바일 과부하).
- 전투 상태 클립: `idle / attack / hit / cast`(+ 승리/패배). 기존 `spriteFx(side,kind)` 호출 자리에서 클립 전환.
- idle = 2~4프레임 느린 루프(평상시).

### 4.3 UI 디자인 토큰
- CSS 변수로 규정: `--px`, 색 램프, 테두리 두께(=`--px`), 라운드(거의 각짐, 0~`--px`), 픽셀 그림자(오프셋=`--px` 배수), 폰트(Galmuri).
- 적용 대상: 버튼·카드·HP바·모달·토스트·전투 이펙트(데미지 팝업/상태 VFX).
- 이펙트도 같은 픽셀 단위·팔레트로(이모지 의존 축소 또는 픽셀 아이콘화).

### 4.4 사운드
- `beep()` 엔진을 **음향 팔레트로 규정**: 파형(square/saw/sine/triangle)·길이·볼륨·음정 세트를 상수화.
- **빠진 이벤트 SFX 채움:** 피격/버프/디버프/단계업/수확/UI 열기·닫기/구매/탐사 등.
- **AI 오디오/BGM:** `assets/audio/`에 두고 씬별(타이틀/탐사/전투) 오버라이드. 파일 없으면 칩튠 폴백. BGM 채널 1개(루프) + SFX 채널.
- `state.sound` 토글 존중(이미 있음). BGM on/off 별도 토글 고려.

## 5. 자산 규약 (`assets/`)
- 구조(제안): `assets/plants/<키>.png`, `assets/plants/<키>__<clip>.png`(스프라이트시트), `assets/audio/sfx/*`, `assets/audio/bgm/*`.
- 등록: 그림 = `SPRITE_OVERRIDES["<타입>_<단계>_<속성>"]`(또는 종 키 스킴), 화분 = `POT_SPRITE_OVERRIDES`, 오디오 = 신규 오디오 맵.
- ⚠️ **`sw.js` 캐시 목록에 신규 자산 경로 반드시 추가** (안 하면 옛것/오프라인 미표시). 자산 추가 = 코드 등록 + sw.js 등록 **2곳 동시**.
- **AI 그림 입력 규격 고정:** 정사각·투명배경(`remove_background`)·피사체 스케일/프레이밍 일치(예: 캔버스 하단 화분 자리, 식물 중심 cx). 안 맞으면 배치·픽셀화 어긋남.

## 6. ⚠️ 함정 모음 (주의할 것)
1. **픽셀 단위 불일치** — 가장 흔한 깨짐. §3 `--px` 규정으로 차단(요소별 임의 픽셀 크기 금지).
2. **sw.js 캐시 누락** — 새 자산이 안 보임/오프라인 실패. 자산마다 등록 체크리스트.
3. **모바일 런타임 픽셀화 과부하** — 정지컷·모션 프레임 모두 **굽고 캐시** 필수. 매 프레임 재픽셀화 금지.
4. **AI 그림 규격 불일치** — 정사각·투명·스케일 고정 안 하면 배치 어긋남.
5. **단일 파일/PWA 정신 유지** — `?dex=1`·preview·`__catalogSelfTest()`·세이브 호환 깨지지 않게. 필터는 순수 함수로.
6. **이모지 혼용** — 도트 톤과 충돌. 점진적으로 픽셀 아이콘으로 대체(우선순위 낮음, 일괄 X).

## 7. 순차 실행 로드맵 (단계별 — 각 단계 독립 검증/커밋)
1. ✅ **규정 확정(2026-06-27)** — `--px:3px` CSS 토큰 + 전역 제한 팔레트 `PIXEL_PALETTE`(7속성 램프+중립 ≈24색, `ELEMENT_PALETTE` 직후) 코드 정의.
2. ✅ **픽셀화 필터(2026-06-27)** — `pixelizeSvg(svg,size,cb)`(SVG→`Image`→캔버스 `round(size/--px)` 다운샘플→`PIXEL_PALETTE` 최근접 양자화→`toDataURL`) + 외형 마크업 해시 캐시. **기존 절차 SVG에 적용** = 자산 0개로 전 식물 도트화. 비동기 굽기는 `MutationObserver` 사후 패스(`applyPixelArt`)로 렌더 경로 밖 분리(`.ss-plant`/`.plant-layer`만 `<img>` 교체, `.ss-pot` 제외, `data-px` 루프차단, `__DEX_MODE`/`state.pixelArt===false` 가드). Chromium 검증: 셀프테스트 0 fail·변이 hue필터 보존·캔버스 오염 없음·화분 미처리·dex 미실행.
3. **UI 토큰화** — 버튼/카드/HP바/모달/이펙트 도트 톤 일괄. ← **다음**
4. **AI 그림 끼우기** — 간판 종부터 `assets/plants/` + 오버라이드 + sw.js 등록.
5. **모션 엔진** — 스프라이트 애니메이터 + 전투 상태 매핑.
6. **사운드** — 칩튠 규정+확장 → AI BGM 오버라이드.

**검증:** 각 단계 후 preview + `__catalogSelfTest()` 반환값(fails 배열)로 회귀 확인. 2단계만 끝나도 이미 전 식물이 도트로 보인다.

## 8. 미해결/후속 결정거리 (구현 중 확정)
- `--px` 실제 값(3px? 디바이스 픽셀비 대응?).
- 픽셀화 다운샘플 해상도 정확값(~96 기준, 식물/UI 별도일 수 있음).
- 종 키 스킴(현 `타입_단계_속성` vs 개체 변이 키)에서 AI 그림 매핑 입도.
- 색 quantize 색 수(16 vs 24)·속성별 팔레트 잠금 정도.
- AI 그림/모션을 어떤 도구로 뽑을지(파이프라인) — 본 설계와 독립.
