## 1. 가치 (왜 만드는가)

현재 전투 콘텐츠는 **봇 토너먼트 무한 반복**뿐 — *목표도 서사도 없다*. 미션 모드는 세 가지 가치를 채운다:

1. **끝이 있는 도전** — 캠페인을 깨면 "클리어"라는 명확한 목표·성취.
2. **아틀라스 세계관 몰입** — 이미 만든 11행성·세력을 무대로 스토리를 입힘.
3. **고유 보상 동기** — 미션에서만 얻는 종으로 수집 욕구 자극.

방치형 양육과 무한 토너먼트 사이의 **"콘텐츠 소비" 축**. PvP/서버(#8)와 독립이라 오프라인에서도 완결된다.

## 2. 진입 & 네비게이션

- 토너먼트 시작 화면 상단에 **세그먼트 탭 `[토너먼트] [미션]`** 추가.
- "미션" 탭 → **캠페인 목록 화면**.
- 미션 모드 자체는 토너먼트창 탭에서 바로 접근(함선 NPC 경유 강제 아님). 함선 NPC가 캠페인을 *소개/해금*하는 건 #4 후속 링크.

## 3. 캠페인 목록 화면

각 캠페인 = 카드. 표시 항목:

| 항목 | 내용 |
|---|---|
| 테마 아이콘 | 행성 또는 인물 |
| 이름 | 예: "볼카르 화산 군락" |
| 난이도 | **★1~5** (적 강함 종합 라벨) |
| 권장 | 생장단계 또는 티어 (예: "권장 성장체 이상 / 실버+") |
| 진행도 | `2/4` + 완료 시 클리어 메달 |
| 잠금 | 해금 조건 미달이면 자물쇠 + 조건 표시 |

**해금 규칙 (확정):** **기본 자유 개방** + **고난도/스토리 후반 일부 캠페인만 선행 클리어 게이트**. 대부분 처음부터 도전 가능(유저가 ★ 보고 알아서 선택), 일부만 "선행 캠페인 클리어" 또는 "티어 도달" 조건.

## 4. 캠페인 내부 = 선형 스테이지

- 진입 → 스테이지 사다리 `1 → N`. 클리어한 데까지 + 다음 1개만 열림.
- 각 스테이지 흐름:
  ```
  [등장 대사(텍스트 박스)] → 단판 전투(고정 적 1기) → 승리 시 보상 + 다음 스테이지 해금
                                                    패배 시 재도전
  ```
- 마지막 스테이지 = **보스**: 단판이되 **스탯 부스트 + 특수 기믹 1개**로 난이도 확보.
- **재플레이:** 클리어한 스테이지 재도전 가능(연습·소액 화폐 파밍). **1회성 보상(캠페인 보상종·첫클리어 보너스)은 재지급 안 됨.**

## 5. 적 설계 (저작된 고정 로드아웃)

플레이어 비례 스케일 봇이 **아니다**. 각 적은 데이터에 박힌 **고정 사양**:
- 종(`species`) · 생장단계(`stage`) · 스킬 로드아웃 · 변이 카드 · 개체 등급(`grade`).
- 기존 `makeCombatant`를 **override 주입**으로 재사용(랜덤 봇 경로 대신 고정 사양).
- 보스 = 위 + **스탯 부스트** + **기믹 1개**.

**보스 기믹 라이브러리 (초기 4~5종):**

| 기믹 키 | 효과 |
|---|---|
| `burn_aura` | 매 턴 플레이어에게 화상 1스택 부여 |
| `shield_start` | 전투 시작 시 보스에 방어막(피해 1회 흡수) |
| `enrage_below_half` | 보스 HP 50% 이하 시 공격력 버프 |
| `thorn_reflect` | 피격 시 받은 피해 일부 반사 |
| `regen_aura` | 보스가 매 턴 소량 자가회복 |

기믹은 기존 상태이상/버프 엔진(`addDot`/`buffs`/반사 패시브) 재활용 — **신규 엔진 최소화**.

## 6. 난이도 모델

- **★ = 소프트 권장.** 적의 절대 강함(스탯·등급·생장·기믹) 종합 라벨 + 권장 티어/생장 표시.
- **막지 않음** — 저티어로 고난도 캠페인 무모하게 도전 가능(유저 자유 선택 의도).

## 7. 보상 (3층 구조)

1. **스테이지 첫클리어** = 크레딧/미네랄 (★ 비례). 재도전 시 소액 또는 없음.
2. **캠페인 완료(보스 격파)** = **미션 전용 변이종 1회성** + 칭호/배지(프로필).
3. **★ 비례 스케일** — 고난도일수록 화폐·보상 등급↑.

- **전용 화폐 안 만듦** — 기존 크레딧/미네랄 재사용(YAGNI).
- **확장 훅(열린 질문):** 보스 시그니처 변이 카드 드롭은 후속(현재 범위 외).

### 7-1. 보상종 조달 = 미션 전용 변이종 재활용 (확정)

새 종을 만들지 않는다. **이미 정의된 `released:false` 변이 140종** 중 캠페인 테마(속성·타입)에 맞는 종을 **캠페인 보스 보상**으로 지급한다.

- 탐사 시그니처 분포(`MUTANT_SIGNATURES`)와 **별개의 획득 경로** — 탐사로도 희귀 발견되지만, 미션 보상은 *확정 1회성* 지급.
- 구현 = 탐사 시그니처가 쓰는 **`released` 우회 패턴 재활용**(`released:false`여도 명시적 지급 경로에선 정상 획득). 봇/일반 풀은 불변.
- 한 변이종이 탐사·미션 양쪽 경로를 가질 수 있음(중복 무해, 수집은 보유 여부만).

## 8. 스토리 전달

- **텍스트 박스**(초상 이미지 optional — 없어도 동작).
- 단위: 캠페인 **인트로/아웃트로** + 스테이지별 **적 등장 멘트** 한두 줄.
- 데이터로 박아 캠페인마다 저작. 도트/초상 에셋은 #3 도트 작업과 연동(후속).
- **함선 NPC 연동(#4 후속):** 함선에서 NPC 대화가 특정 캠페인을 *소개·해금*하는 트리거 — 미션 모드 1차 범위 외, 링크만 열어둠.

## 9. 데이터 모델

```js
// 캠페인 정의 (코드 인라인 상수)
const CAMPAIGNS = [{
  id: 'volkar_caldera',
  name: '볼카르 화산 군락',
  theme: { kind: 'planet', planet: 'volkar', icon: '🌋' },
  stars: 3,                         // ★ 난이도
  recommend: { stage: 'growth', tier: 'silver' },
  unlock: null,                     // null=자유개방, {clear:'<campaignId>'} 또는 {tier:'gold'}=게이트
  story: { intro: '...', outro: '...' },
  stages: [
    { enemy: { species:'...', stage:'...', loadout:[...], cards:[...], grade:'B' },
      intro: '적 등장 멘트',
      reward: { credit: 120, mineral: 0 } },
    // ...
  ],
  boss: { enemy: {...}, statBoost: 1.25, gimmick: 'burn_aura',
          intro: '보스 등장 대사' },
  clearReward: { speciesKey: '<released:false 변이종 key>', title: '화산 정복자' }
}];

// 진행 상태 (state에 저장 + normalizeState 마이그레이션)
state.mission_progress = {
  [campaignId]: { cleared: [0,1], done: false }  // cleared=클리어한 스테이지 인덱스, done=캠페인 완료
};
```

- `normalizeState`에 `state.mission_progress ??= {}` 마이그레이션(무회귀).
- 보상종 지급 = 기존 종자 인벤토리 추가 경로 재사용, `released` 우회.

## 10. 초기 콘텐츠 목표

캠페인 **3~4개**로 출발, 데이터 추가만으로 확장:
- **볼카르 불 캠페인** (★3, 불 속성 적단)
- **네레이돈 물·빙결 캠페인** (★4, 물/빙결 적단)
- **인물: 전 챔피언 캠페인** (★5, 선행게이트 — 다른 캠페인 클리어 후 해금)
- (확장) 추가 행성 테마

## 11. 구현 단위 (writing-plans 입력용 경계)

각 단위는 독립 테스트 가능하게 분리:

1. **데이터 레이어** — `CAMPAIGNS` 상수 + `state.mission_progress` + `normalizeState` 마이그레이션 + 보상종 지급 헬퍼(`released` 우회).
2. **고정 적 인스턴스화** — `makeCombatant` override 주입(고정 로드아웃·등급·생장), 보스 스탯부스트+기믹.
3. **기믹 엔진** — 보스 기믹 4~5종(기존 상태이상/패시브 재활용), `__test` 케이스.
4. **전투 진입/종료 배선** — 단판 전투 시작→승패→진행도 갱신→보상 지급→다음 해금. 재플레이 시 1회성 보상 차단.
5. **UI: 캠페인 목록** — 탭 추가, 카드(★·권장·진행도·잠금) 렌더, 해금 게이트 표시.
6. **UI: 캠페인 내부** — 스테이지 사다리, 등장 대사 텍스트 박스, 보상 연출(기존 `openRewardReveal` 재활용).
7. **검증** — `__catalogSelfTest()` 케이스(캠페인 데이터 정합·보상종 실재·진행도 마이그레이션) + preview 실플레이.

## 12. 범위 밖 (후속 / 열린 질문)

- 보스 시그니처 변이 카드 드롭(확장 훅).
- 함선 NPC 스토리 해금 트리거(#4 연동).
- 초상/도트 에셋(#3 연동, 현재는 텍스트만).
- 캠페인 별점 3성 클리어 등 *도전 과제* 레이어(YAGNI, 후속).
- 미션 전용 화폐·상점(도입 안 함 확정).

## 13. 확정 결정 요약

| 항목 | 결정 |
|---|---|
| 진행 구조 | 캠페인 선택형(★ 라벨) + 내부 선형 고정 적 |
| 해금 | 기본 자유 + 고난도 일부 선행게이트 |
| 전투 형식 | 전부 단판 (보스 = 단판 + 스탯부스트 + 기믹) |
| 적 | 저작된 고정 로드아웃 (스케일 봇 아님) |
| 난이도 | 소프트 권장(막지 않음) |
| 보상종 | 미션 전용 = `released:false` 변이 140종 재활용 |
| 전용 화폐 | 없음 (크레딧/미네랄 재사용) |
| 스토리 | 텍스트 박스(초상 optional), 캠페인 인트로/아웃트로 + 스테이지 멘트 |
