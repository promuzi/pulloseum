# 개체 외형 전면 재제작 — 픽셀 아트 개별 스프라이트 설계 (2026-06-27)

> 상태: **인프라 구현 + 덩굴 42/42 완성(2026-06-27)** · 파일럿 = 덩굴(vine) 묶음 · 버섯은 개체 추가 작업 완료 후 합류
> → 구현 현황은 맨 아래 **§10**.

## 1. 배경 / 문제

현재 식물 외형은 `composePlantSvg` → `composePlantBody`(타입×단계×속성 실루엣) 위에 **`featuresSvg`(눈·뿔·날개·더듬이를 키 해시로 랜덤 부착)** + `markingSvg`(무늬)를 덧대는 절차적 합성이다. 이 **랜덤 부속 레이어(`FEATURE_MODULES`)가 지저분·징그럽고 정신사납다**는 것이 사용자 지적. 또한 "같은 타입+속성은 색·필터만 달라" 개체 고유성이 약하다.

→ **개체마다 컨셉에 충실한 고유 외형을 새로 그린다.**

## 2. 확정 원칙 (브레인스토밍 결과)

- **매체 = 픽셀 아트(도트), 고해상(~42×54칸급, 식물 본체 기준).** 코드 내장 ASCII 격자 → `pixelArt(rows,pal,cell,ox,oy)`(이미 존재, `POT_PIXELS`/화분이 같은 방식)로 SVG rect 렌더. 외부 파일 없음, 도감·전투 자동 반영. (24×30급은 갑옷 띠·가시 등 디테일이 뭉개져 식별 어려움 → 42×54로 상향 확정, 2026-06-27.)
- **🚫 색·효과만으로 개체를 구분하지 않는다(하드 원칙).** 두 개체가 "같은 격자, 다른 색"이 되는 일은 절대 금지. **정체성은 항상 모양(격자)이 짊어진다.** 175 개체 = 175개의 서로 다른 격자.
- **속성(element) = 그 개체 *자기 속성* 색을 입히는 팔레트 역할만.** 팔레트 교체로 자동 리컬러(기존 시스템의 장점 유지)하되, 이는 *개체 구분 수단이 아님* — 개체는 이미 격자가 다르다.
- **개체당 대표 픽셀 1장(성체 기준).** 175장. 생장 단계는 엔진이 크기·약간의 가감으로 처리, 씨앗·새싹·유체(초반 3단계)는 공용 단순형 유지. 단계별 고유 격자는 추후 별도 패스(YAGNI).
- **랜덤 부속(`featuresSvg`)·무늬(`markingSvg`)는 픽셀 격자가 있는 개체에서 제거(미사용).** 전 종 커버 후 함수 자체 정리.

## 3. 화풍 / 해상도

- **셀셰이딩 톤**: 격자 안에 **외곽선 문자(`O`)** 를 직접 찍어 진한 테두리를 만들고, 본체는 메인/암부/명부 2~3톤. 작은 크기(전투 108px)에서도 실루엣이 또렷하게.
- **해상도**: 식물 본체 영역 **~42×54칸**, `cell`은 120×140 viewBox 기준 식물 영역(대략 폭 70·높이 100)에 맞춰 산정(예: cell≈1.7~2). 화분(`POT_PIXELS` cell=5)보다 촘촘 — 식물 정체성(갑옷·가시·아가리 등)이 또렷이 읽혀야 하므로. 파일럿에서 최종 `cell`·정렬 확정.
- **정렬**: 기존 좌표 규약 유지 — base y96(화분 입구)에 식물 바닥 정렬, 중심 x=60. `ox/oy`로 격자를 이 기준에 앉힌다.

## 4. 데이터 / 렌더 구조

### 4.1 종별 픽셀 레지스트리
```js
// 키 = 종 키(SPECIES_CATALOG 키). 값 = { rows:[...ascii], cell?, ox?, oy? }
const SPECIES_PIXELS = {
  venom_weed: { rows:[ "....", ... ] },   // 예: 덩굴/독성
  // ...
};
```

### 4.2 팔레트 문자 역할 (격자 char → 색)
- **속성 연동(개체 자기 속성으로 리컬러)**: `B`=본체 메인, `b`=본체 암부, `l`=본체 명부, `F`=속성 기운 메인, `f`=속성 기운 명부 → `elementPalette(el)`에서 매핑(`P.main/dark/light/accent` 등).
- **구조 고정색(속성 무관)**: `O`=외곽선(아주 어둡게), `S/s/H`=금속/갑옷(강철·암부·하이라이트), 필요 시 `W`=목질, `K`=뿔/이빨(상아) 등. 이 글자는 모든 속성에서 같은 색.
- 팔레트 합성: `const pal = Object.assign({}, STRUCT_PALETTE, elementBodyPalette(el))`. (데모에서 검증된 모델 — 갑옷 금속색 고정 + 본체/기운만 속성색)

### 4.3 `composePlantSvg` 분기
- 현 시그니처 그대로(`opts.varKey`=종 키, `opts.form`=변이형). 내부에서:
  1. `gi = stage index`. **`SPECIES_PIXELS[opts.varKey]`가 있고 `gi>=3`(성장체↑)** 이면 → 픽셀 렌더 경로: `pixelArt(rows, pal, cell, ox, oy)` + 속성 모티프(`elementMotif`)만 선택적으로 얹고, **`markingSvg`/`featuresSvg`는 건너뜀**.
  2. 아니면 → 기존 `composePlantBody`(+필요 시 기존 레이어) 폴백.
- 단계 가감: `gi===4`(성체)=기준 격자, `gi===3`(성장체)=축소·일부 생략, `gi===5`(완숙체)=오라/포자 등 기존 `aura`/`spark` 연출 유지(격자 위에 얹음). 초반(gi<3)은 기존 공용 실루엣.
- **흔들림 모션**은 식물 SVG 전체에 걸리므로 픽셀에도 그대로 적용됨.

### 4.4 PNG 훅 공존
- 기존 `SPRITE_OVERRIDES`(PNG, 종.단계 키)는 그대로 — 사용자 손그림 PNG가 있으면 그것이 최우선(현 우선순위: PNG override > 픽셀 격자 > 절차적). 픽셀은 그 사이 기본값.

## 5. 진행 계획 (묶음 = 타입)

> 묶음마다 **구상 → 픽셀 → 게임 반영 → 사용자 확인** 1사이클. 파일럿으로 화풍·`cell`·정렬·팔레트 역할을 **확정**한 뒤 같은 틀로 양산.

| 순서 | 묶음(타입) | 비고 |
|---|---|---|
| 0 (파일럿) | **덩굴 vine** | "갑옷 덩굴" 등 코일 실루엣으로 파이프라인·화풍 확정 |
| 1 | 다육 cactus | 구조 단순(몸통+가시+팔) |
| 2 | 화초 flower | 꽃잎 실루엣 다양 |
| 3 | 목본 tree | 수관 실루엣 |
| 4 | **버섯 mushroom** | ⚠️ **개체 추가 작업 완료 후 합류**(현재 보류) |

- 개체 수는 plan 단계에서 `SPECIES_CATALOG`를 타입별로 열거해 확정(기본 35 + 변이 140 = 175, 단 버섯 제외분부터).

## 6. 개체 구상 워크플로 (개체당 하나씩 빠르게)

각 개체에 대해:
1. **컨셉 읽기**: `SPECIES.desc`(컨셉문, `applyVariantIdentity` 생성) + `archetype`(14종 성격) + `baseVariants`(변이형) + 고유스킬 이름(`ind.<key>.*`) + 타입·속성.
2. **한 줄 외형 브리프** 작성: 예) `venom_weed(덩굴·독성·풀) → "독액 맺힌 가시 코일 덩굴, 끝에 벌어진 독주머니"`.
3. **픽셀 격자 1장** 제작(성체 기준, 셀셰이딩 톤, base y96 정렬). 변이형은 실루엣에 반영(무기=갑옷/날, 포식=아가리/이빨, 독성=가시/독주머니, 용족=뿔/비늘/날개, 일반=정연).
4. 레지스트리 등록 → 게임 반영.

## 7. 검증

- `window.__catalogSelfTest()` 반환 fails 배열로 회귀 확인. **케이스 추가**: ① 픽셀 등록 종은 `gi>=3`에서 `featuresSvg`/`markingSvg` 산출이 비어야 함(랜덤 부속 미혼입), ② 같은 종을 두 속성으로 렌더 시 *구조 고정색 글자(O/S 등) 픽셀 위치는 동일*하고 *본체색만 다름*(격자=정체성 불변 검증), ③ 픽셀 미등록 종은 기존 폴백 정상.
- preview(`pullosseum`)로 도감/전투 108px·78px 크기에서 또렷함 육안 확인. `preview_screenshot`은 배경 애니로 타임아웃 잦음 → `preview_eval`로 SVG DOM(rect 수·색)·`composePlantSvg` 반환 문자열 확인 우선.
- 묶음 완료 시 도감(`docs/dex/plant-codex.html`) 자동 반영 확인(데이터 복제 없음).

## 8. 미해결 / 추후 (YAGNI 분리)

- 단계별 고유 격자(성장체/완숙체 전용 그림) — 1바퀴 완성 후 별도 패스.
- 초반 3단계(씨앗·새싹·유체) 종별 고유화 — 현재 공용 유지.
- 전 종 커버 완료 후 `FEATURE_MODULES`/`markingSvg`/`individualFeatures` **죽은 코드 정리**(선언 스텁 보존 규약 준수).
- 버섯 묶음은 개체 추가 작업 완료 신호 후 동일 틀로 합류.

## 9. 결정 로그 요약

- 매체: **픽셀 아트(중해상도)** — 도트 게임 정체성·엔진 적합(`pixelArt` 기존)·속성 자동 리컬러 유지.
- 개체 구분: **모양(격자)만으로**. 색·효과만의 변형 금지(하드).
- 분량: 개체당 1장(성체), 175장, 단계는 엔진 가감.
- 진행: 타입별 묶음, 덩굴 파일럿, 버섯 보류.
- 통합: `SPECIES_PIXELS` 레지스트리 + `composePlantSvg` 분기 + 절차적 폴백(점진 적용).

## 10. 구현 현황 (2026-06-27)

**✅ 인프라 구현 완료:**
- `STRUCT_PALETTE`(O 외곽 `#1b2436`·S/s/H 금속·W/w 목질·K/k 상아 — 속성 무관) + `elementBodyPalette(el)`(B/b/l 본체·F/f 기운=속성 팔레트) + `speciesPixelPalette(el)` 합성.
- `SPECIES_PIXELS` 레지스트리 + `renderSpeciesPixels(pix,el,gi)`(자동 정렬: 바닥 base y96·중심 x=60, gi3=축소). 기존 `pixelArt(rows,pal,cell,ox,oy)` 재사용.
- `composePlantSvg` 분기: `opts.varKey` 등록 종 + `gi>=3` → 픽셀 경로(`featuresSvg`/`markingSvg`/variant 색필터 **스킵**), 그 외 기존 폴백. 픽셀 경로 svg에 `data-pixelgrid` 표식.
- **런타임 픽셀화(unification 폴백)와 공존:** 사후 패스 `applyPixelArt`가 `data-pixelgrid` 격자는 **재픽셀화 스킵**(블러 방지). 손그림 격자 = 정체성 / 런타임 픽셀화 = 아직 안 그린 종 임시 도트 폴백.

> ⚠️ **용어:** 비버섯 종의 **변이형 = 5개**(일반·포식·무기·독성·용족). 포자(spore)=버섯 전용, 발광(lumen)=폐지라 제외. 여기에 base(비변이)를 더해 덩굴은 6개 그룹 × 7속성 = 42.

**✅ 덩굴 5종 — 6개 그룹(base + 5 변이형) 중 5개 실루엣 템플릿 확정(화풍 사용자 승인):**
- 고해상 **cell=1.8(~22×42칸)**. 변이형이 실루엣을 결정 → 개체가 모양으로 구분(정체성=모양 검증), 속성은 리컬러만(구조색 불변·본체색만 변경 실측).
  - `thorn`(base/풀) = 잎+가시 / `venom_weed`(toxic/풀) = 가시 줄기+열린 독주머니 / `carni_vine`(pred/불) = 이빨 포충낭(아가리) / `whip_vine`(weapon/불) = 칼날+미늘 채찍 / `draca_flame`(dragon/불) = 뿔+막날개+비늘띠.
- 셀프테스트 케이스 2종(픽셀 경로/폴백·속성 리컬러 불변) — `__catalogSelfTest()` **0 fail**.

**✅ 배치 2 — normal 템플릿 + base 7속성 완성:** `green_vine`(normal/풀=무성한 둥근 잎) + 기본 덩굴 6속성(`vine_fire` 불꽃잎·`vine_water` 물방울잎·`vine_earth` 넓은잎·`vine_wind` 휜잎·`vine_bolt` 번개잎·`vine_ice` 결정잎). **속성별 잎 모양을 다르게**(색만 구분 금지 준수). 6개 그룹 템플릿 전부 확보(base + 5 변이형: toxic·pred·dragon·weapon·normal). 셀프테스트 0 fail.

**✅ 배치 3 — weapon 7속성 완성:** `whip_vine`(칼날채찍)·`kelp_whip`(넓은 켈프날)·`thorn_whip`(가는 외날)·`root_club`(옹이 곤봉)·`wind_whip`(갈래 채찍)·`volt_whip`(갈래 번개날)·`frost_whip`(각진 결정날). 무기형 안에서도 곤봉/채찍/칼날 실루엣 차별.

**🔜 다음:** 덩굴 나머지 6종(normal 각 6 + 속성·아키타입 변주) → 타입별 묶음(다육→화초→목본→버섯). 단계별 고유 격자·초반 3단계·죽은 `FEATURE_MODULES` 정리는 1바퀴 후(YAGNI §8). **진행: 덩굴 42/42 완성 ✅.**
