# 전투 UI 개편 설계 (2026-06-25)

> 상태: **설계 완료 · 코드 미반영**. 다음 단계 = 구현 플랜(writing-plans).
> 접근: **A — 기존 요소 용도변경**(새 DOM 골격 X). 단일 `index.html` + 테스트 러너 없는 환경의 회귀 위험 최소화.

## 1. 배경 / 문제

현재 전투는 매 턴 카드가 중앙에 떠올랐다 사라지고(`#cardPhase` 오버레이), 공격 1회당 여러 문구가 순차로 떠서(`치명타!` → 효과 텍스트 → 데미지 → 상태이상 메시지 → 지속피해) 길고 가독성이 떨어진다. 사용자 불만 3가지:

1. 카드가 매번 등장/퇴장 → 정신 사나움
2. 판정 문구·모션이 너무 길고 가독성 나쁨
3. 카드 UI가 복잡함(아이콘·이름·비용·설명 다 노출)

## 2. 목표 (확정 결정)

전투 화면을 **포켓몬식 고정 레이아웃 + 간결 판정 연출**로 개편한다.

- 스킬은 **하단에 항상 고정**(매 턴 등장/퇴장 폐지). 선택 가능할 때만 불 켜짐, 아니면 꺼짐.
- 식물은 **상단**에 포켓몬식 대각 배치(적 위, 나 아래) + 각자 체력/에너지 바.
- 판정 시 **상단(식물 영역)만 흐려지고** 그 위에 양쪽 카드가 등장.
- 데미지/상태 적용 순간엔 **흐림 해제**(양쪽 HP가 보여야 함).
- 공격 1회 = **상성 한 줄 + 데미지 숫자 + 상태 VFX 한 번**, 순차 메시지 폐지.
- 카드 앞면 = **이름(상) / 아이콘(중) / 분류·비용(하)** 미니멀, 상세는 **꾹→뒤집기**.
- `battleViewBtn`(스킬 숨기기/보이기 토글) **폐지**.

## 3. 레이아웃

```
┌─────────────────────────────────┐
│ [적]🌿  ▓▓▓ HP   ⚡⚡⚡            │  A. 상단의 상단: 적(좌) + HP/에너지
│                                 │
│            HP ▓▓▓  ⚡⚡⚡   🌱[나] │  A. 상단의 하단: 나(우) + HP/에너지
├─────────────────────────────────┤
│  🃏   🃏   🃏                     │  B. 하단 고정 스킬바
│  🃏   🃏   🃏                     │     일반 3열×2행 (cp-hand 그대로)
│       🃏   🃏                     │     변이 1~2칸 (cp-pred 그대로)
└─────────────────────────────────┘     에너지 ⚡ 띠는 스킬바 위 얇게
```

- **A. 상단(식물 무대)** = 기존 `#battleArena`. 적 `#enemyFighter`(위)·나 `#playerFighter`(아래) 유지. `#judgeWindow`는 가운데 끼우는 창이 아니라 **이 영역 위에 떠오르는 판정 오버레이**로 역할 변경.
- **B. 하단(고정 스킬바)** = 기존 `#cardPhase`/`.cp-stack`를 **오버레이→영구 고정 풋바**로 CSS 재배치. 마크업(`#cardEnergy`/`#cardHand`/`#cardPred`)·3열 그리드 그대로 재활용.

## 4. 컴포넌트별 설계

### 4.1 하단 고정 스킬바 (`#cardPhase` → footer 고정)

- 위치: `position` 고정 풋바, 전투 화면 하단. 매 턴 `show/hideCardPhase` 호출 폐지 — 전투 시작 시 1회 렌더 후 상주.
- **불 켜짐/꺼짐 상태**:
  - *내 선택 차례 & 사용 가능*(에너지/횟수 충분) → 불 켜짐(클릭 가능).
  - *사용 불가*(비용 부족·횟수 소진) → 꺼짐(현 `skillUnusable` 재사용, 회색).
  - *판정 진행 중*(`B.busy`) → 전체 꺼짐(클릭 차단).
- 에너지 변동·턴 전환 시 스킬바만 부분 갱신(`refreshSkillBar()` 신설 — 기존 `showCardPhase` 렌더 로직 분리).
- 변이 스킬(`cp-pred`) 1~2칸은 3행째에 그대로.

### 4.2 카드 앞면 (`skillCardHtml` 재구성)

세로 3단:

```
┌──────────────┐
│   스킬 이름     │  상: displaySkillName(name)
│      🔥       │  중: 대표 아이콘(s.icon 이모지, 크게)
│  ⚔️🔥🟣   ⚡2  │  하: 분류칩(skillCatChipsHtml) + 비용 배지(moveCostLabel)
└──────────────┘
```

- **설명(desc)·계수(coefLabel) 제거** — 앞면엔 없음.
- 하단 분류칩: 기존 `skillCatChipsHtml(sk)`+`CAT_META` 재사용 → `cats`(공격/방어/버프/디버프/체력회복/에너지회복) 아이콘. 속성칩(`ELEMENTS[s.elem]` 아이콘, `s.elem`일 때)·독계열칩(`s.dot`/`s.infuse` kind→🟣/🩸/🔥)도 같은 칩 줄에 추가.
- 비용 배지: `moveCostLabel(s, unit)` → `⚡N` 또는 `🔁N회`, 우하단 작게.
- 등급은 기존대로 카드 테두리 색(`gradeColor(battleSkillGrade())`).

### 4.3 꾹 누르기 → 카드 뒤집기 상세

- 트리거: 기존 `bindSkillCards`의 롱프레스(480ms) 유지.
- 연출: 꾹 누른 카드가 **앞으로 확대되며 Y축 회전(rotateY)으로 뒤집힘** → 뒷면 표시. (배경 살짝 어둡게, 아무 곳 탭하면 닫힘 — 현 `#battleSkillDetail` 흐름 재사용.)
- 뒷면 내용 = 기존 `showSkillDetail` 그대로: 이름·비용·태그·**계수 포함 전체 설명**(`skillCoefDesc`). 즉 "자세한 데미지 계수"는 뒷면에서 확인.
- 구현: `#battleSkillDetail` 카드에 flip 트랜지션 CSS 추가. 내용 함수는 그대로.

### 4.4 판정 오버레이 (`openJudge`/`playerSkill` 재배선)

**한 턴 시퀀스:**

1. 하단 스킬 불 켜짐 → 플레이어 카드 탭. (`B.busy=true` → 즉시 스킬바 전체 꺼짐)
2. 적 카드 선택(`aiPickSkill`).
3. **양쪽 카드가 상단(식물 무대) 위로 등장** + **상단 흐림 ON**. 식물 영역에 `backdrop-filter: blur()` 또는 블러 레이어. 카드는 적 카드(상)·내 카드(하) 좌우/상하로.
4. **선공 판정**: 기존 로직(`isPriorityDefenseSkill` 방어 우선 → 아니면 `spd` 기반 `firstSide`). 표시는 간단히 — "선공"/"방어 우선" 짧은 라벨 1개(`setJudgeOrder`).
5. **후공 카드는 물러나고(페이드/축소), 선공 카드만 중앙으로 이동.**
6. 선공 카드 → **상성 한 줄**: `eff(attacker.element, defender.element)` 배수로 결정.
   - `>1` → "효과가 굉장하다!" / `<1` → "효과가 별로…" / `=1` → "정확히 들어갔다!" (치명타면 같은 줄에 빨강 "치명타!" 병합).
7. **흐림 해제 ON→OFF** → `applySkill`이 **모든 효과(피해·독·버프·디버프)를 한 번에 적용**:
   - 데미지 숫자 `popup()` + 상태 VFX 번쩍(§4.5). 순차 메시지(`showEffectMessages`)·`치명타!` 별도 줄 **폐지**.
8. **후공 카드 등장 → 중앙 이동 → 상성 한 줄 → 흐림 해제 → 데미지/VFX** (6~7 반복).
9. 턴 끝 지속피해/디버프 정리(`tickStatuses`)도 **동시에** — 텍스트 줄 없이 데미지 숫자 + VFX 번쩍 + 상태바 아이콘 갱신.
10. 카드 정리, 에너지 +1, `B.turn++`, `B.busy=false` → 하단 스킬바 다시 불 켜짐.

- 방어 스킬은 기존대로 공격보다 먼저 적용(우선) — 시퀀스에선 "선공 카드"가 곧 방어카드일 수 있음.
- 사망 시 기존 `finishMatch` 흐름 유지.

### 4.5 상태 VFX (순간형, 식물 스프라이트 위)

- **순간형만**: 걸리거나 피해 들어오는 그 순간에 1회 번쩍 후 사라짐. **지속 표시는 안 함** — 유지 상태는 기존 상태바 아이콘(`statusTags`: 🧊보호막/독·출혈·화상 아이콘/🔺버프/🔻디버프)으로 충분.
- 트리거 위치: `#pSpriteBox`/`#eSpriteBox` 위에 일시 VFX 엘리먼트를 띄웠다 제거(현 `popup`과 동일 패턴).
- 매핑(자유 재량, 초안):
  - **방어/보호막 적용** → 금속 코팅막이 식물을 감싸는 번쩍.
  - **보호막/방어 깨짐**(barrier 0으로 소진) → 코팅막이 쩍 갈라지는 연출.
  - **독 적용·중독 피해** → 보라색 한 번 번쩍 후 사라짐.
  - **화상 적용·화상 피해** → 불 붙는 번쩍.
  - **출혈** → 붉은 방울 튐.
  - **버프** → 상승 화살/오라, **디버프** → 하강 그림자. (둘 다 짧게)
- 신설 함수 1개: `spriteFx(side, kind)` — kind에 따라 클래스 부여 후 타임아웃 제거. CSS 키프레임으로 연출.

### 4.6 폐지 항목

- `battleViewBtn`(`toggleBattleView`·`B.view`·`showBattleField`/`renderBattleIdleView`의 view 분기) — 스킬 상시 고정이므로 제거. `renderBattleIdleView`는 "스킬바 불 켜기"만 담당하도록 단순화.
- 공격당 순차 텍스트 체인(`showEffectMessages`·중복 `showJudgeMessage` 호출·별도 `치명타!` 줄) — 상성 한 줄로 대체.

## 5. 재사용 자산 (신규 최소화)

| 용도 | 기존 함수/요소 | 비고 |
|---|---|---|
| 상성 배수 | `eff(elemA, elemB)` | applySkill 내 `elemMult` 동일 |
| 속성 아이콘/색 | `ELEMENTS[elem]` | |
| 분류 아이콘 | `skillCatChipsHtml` + `CAT_META` | 카드 하단칩 |
| 비용 라벨 | `moveCostLabel(s,unit)` | |
| 사용 불가 판정(불 끄기) | `skillUnusable(unit,id)` | |
| 등급 색 | `battleSkillGrade`+`gradeColor` | 테두리 |
| 뒷면 상세 내용 | `showSkillDetail` / `skillCoefDesc` | 그대로 |
| 지속 상태 아이콘 | `statusTags(u)` | 상태바 유지 |
| 데미지 숫자 | `popup(side,text,isHeal)` | |
| 카드 그리드 | `.cp-hand`(3열)·`.cp-pred` | 고정바로 재배치 |

**신규**: `refreshSkillBar()`(스킬바 부분 렌더/불 상태), `spriteFx(side,kind)`(상태 VFX), 카드 flip CSS, 상단 블러 레이어 CSS, `skillCardHtml` 앞면 3단 재구성.

## 6. 회귀 검증

- `window.__catalogSelfTest()` 반환값(fails 배열)로 판정. 기존 `cats`/카드효과 테스트 깨지지 않는지 확인.
- 수동: 미리보기에서 (1) 스킬바 상시 고정·불 켜짐 토글, (2) 카드 탭→양쪽카드 상단 등장→상단 흐림→선공 카드 중앙→상성 한 줄→흐림 해제+데미지, (3) 꾹→뒤집기 상세, (4) 독/방어/화상 VFX 번쩍, (5) 항복·승패·다음 턴 정상.
- preview는 정적 서버(HMR 없음) → 수정 후 `location.reload()`, 안 풀리면 `preview_stop`+`preview_start`. 스크린샷은 배경 애니로 타임아웃 잦으니 `preview_eval`로 DOM 검증 병행.

## 7. 범위 밖 (YAGNI)

- 스킬 아이콘을 이모지 → 새 일러스트로 교체(별도 큰 에셋 작업, 훅만 열어둘 수 있으나 이번 범위 X).
- 전투 밸런스/데미지 공식 변경 없음(연출/표시만 개편).
- 상태 VFX의 지속형(상시 레이어) — 순간형으로 확정, 지속은 아이콘.
