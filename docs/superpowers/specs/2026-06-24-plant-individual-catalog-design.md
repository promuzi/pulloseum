## 0. 배경 / 목표

현재 종 시스템은 **속성 1개 × 타입 1개 = 35종을 절차적으로 자동 생성**한다(`SPECIES_GRID` 한 줄 → 스탯·외형·스킬 자동). 스킬은 속성+생장단계 풀(`ELEMENT_GROWTH_SKILLS`)에서 공유되고, 변이형은 카드로 후천 부여된다.

**새 방향:** 자동 격자를 넘어, **개체(species) 하나하나를 손으로 정의하는 카탈로그**로 전환한다. 같은 타입·생장단계여도 **스킬·스탯·변이 슬롯이 다른 별개 개체**를 여러 개 만들어 수집의 다양성을 확장한다. 핵심 요구는 **"수정·확장이 편한 데이터 구조"** — 향후 스킬 상세를 따로 작업할 때 편하게 고칠 수 있어야 한다.

**제약(불변):** 빌드 없는 **단일 파일 `index.html`**, `file://` 더블클릭 실행. 외부 데이터 파일 `fetch`는 `file://`에서 CORS로 막히므로 **금지** → 모든 데이터는 인라인.

## 1. 데이터 구조 — A+C (인라인 + 정의/배정 분리)

### 1-1. 개체 카탈로그 `SPECIES_CATALOG`

`SPECIES_GRID`(한 줄짜리)를 대체/보강한다. **개체 1개 = 객체 1개.**

```js
{
  key:'blazebloom',                // 고유 키 (세이브 호환 — 기존 7키 유지)
  name:'블레이즈블룸', type:'flower', element:'fire',
  rarity:'common',                 // 등장/드롭률만 좌우 (스탯 영향 미미)
  stats:null,                      // null → type+element 자동스탯 / 객체 → 개체 고유 스탯
  variantSlots:{ normal:2, weapon:3 },  // 변이 슬롯 배분 (개체마다 1:4, 2:3 …)
  baseVariants:['weapon'],         // 태생 기본 변이 (버섯은 ['spore'])
  stageSkills:{                    // 생장단계별 해금 스킬 (공통키 + 고유키 혼합)
    seed:[], sprout:['common.fire.ember','sig.weapon_ignite'],
    juvenile:[…], grown:[…], adult:[…], mature:[…]
  },
  signatures:['weapon_ignite']     // 개체 고유(시그니처) 스킬 키
}
```

- 명시하지 않은 값(`stats:null`, 외형 미지정)은 **기존 자동생성으로 폴백** → 점진 전환 가능, 더블클릭 유지.
- 기존 `SPECIES_GRID`/`SPECIES`/파생 맵(`SPECIES_SEEDTYPE`, `SPECIES_BY_TYPE_EL` 등)은 카탈로그에서 **자동 파생**하도록 어댑터를 둬 기존 호출부 무회귀.

### 1-2. 스킬 정의 분리 `SKILL_LIB`

스킬의 **실제 효과 정의는 한 곳**(`SKILL_LIB`), 개체는 **키로 참조만** 한다.

```js
SKILL_LIB = {
  'sig.weapon_ignite': { name:'무기 화염화', desc:'3턴간 무기 스킬에 화염 부여', effect:{…} },
  'common.fire.ember': { name:'…', effect:{…} },
  'mushroom.spore_burst': { name:'…', effect:{…} }   // 버섯 전용 네임스페이스
}
```

**효과:** 개체 추가 = 카탈로그 객체 한 블록 / 스킬 수정 = `SKILL_LIB` 한 곳. 둘이 독립적으로 편집 가능. 기존 `ELEMENT_SKILLS`/`ELEMENT_GROWTH_SKILLS`는 `SKILL_LIB`로 흡수하거나 어댑터로 참조.

### 1-3. 키 네임스페이스 규약

- `common.<element>.<name>` — 속성/단계 공통 스킬
- `sig.<name>` — 개체 시그니처(변이 연동 포함)
- `mushroom.<name>` — 버섯 전용 메커니즘
- 변이형 키는 기존 유지: `normal/predator/weapon/toxic/spore/glow/dragon`

## 2. 타입 개편 — 초본형 → 버섯형

5타입 유지. **초본형(grass type) 제거, 버섯형 추가.**

| 타입 | key | 성향 | 상태 |
|---|---|---|---|
| 🍄 버섯형 | `mushroom` | **저스탯 + 포자 기본** | **신규** |
| 🌳 목본형 | `tree` | 체력·방어 | 유지 |
| 🌸 화초형 | `flower` | 공격·화력 | 유지 |
| 🌵 다육형 | `cactus` | 방어·지구력 | 유지 |
| 🍃 덩굴형 | `vine` | 기동·흡혈 | 유지 |

> 제거 사유: 초본형(풀)과 화초형(꽃)의 정체성이 겹친다.

### 2-1. 마이그레이션 — (b) 레거시 외형 전용 보존

- 초본형은 **새 카탈로그에 넣지 않는다.** 하지만 `TYPE_STATS['grass']`와 초본형 외형 생성 코드(`composePlantBody`의 초본형 실루엣)는 **남겨 둔다.**
- 기존 세이브의 초본형 개체(**플레임모스 `flame`, 윈드리프 `windy`**)는 타입을 바꾸지 않고 **그대로 렌더링·전투**된다 → 세이브 무회귀.
- 즉 초본형은 "레거시 전용": 신규 획득 경로에선 안 나오고, 기존 보유분만 유지.

## 3. 버섯형 특수 규칙

- **저스탯:** `TYPE_STATS['mushroom']`을 모든 타입 중 가장 낮게(초안: HP 40 / 공 9 / 방 5 / 기동 8 — 구현 시 밸런스 시트에서 확정). 포자 메커니즘으로 보완.
- **포자 변이 기본 탑재:** 버섯 개체는 `baseVariants:['spore']` 고정. **그 외 모든 식물은 포자 변이 없음.**
- **확장 여지(중요):** "포자 = 버섯 전용"을 코드에 하드코딩하지 않는다. 포자 보유 여부는 **개체별 `baseVariants` 데이터로만** 결정 → 나중에 다른 종에 `'spore'` 한 줄 추가하면 됨.
- **희귀 획득:** `rarity`를 높게 설정 → 등장률·드롭률 낮춤. 탐사 분포에서 특정 지역/조건에서만 출현.
- **별개 메커니즘/스킬:** 버섯 전용 스킬은 `SKILL_LIB`의 `mushroom.*` 네임스페이스로 격리, 기존 속성 스킬 풀과 섞지 않음.

## 4. 변이 슬롯 & 희귀도

### 4-1. 변이 슬롯

- 개체마다 `variantSlots`로 칸 배분(합계는 기존 로드아웃 한도 내). 변이 2개 이상이면 칸이 나뉨(예 1:4, 2:3).
- **변이 연동 고유 스킬:** 시그니처 스킬(예: 무기 화염화)은 해당 변이 슬롯 장착 시 활성.

### 4-2. 희귀도 `rarity`

- 종(개체) 속성. **등장률·드롭률만** 차등. **스탯 차이는 작게** 유지(밸런스).
- 희귀도↑ → **시그니처 스킬 수↑ + 변이 슬롯 다수 허용.**
- **개체 내 품질 차이는 없음**(같은 개체는 항상 동일 — 개별 유전자 품질 등급 도입 안 함).

---

## 5. 스킬 분담

- **공통 + 고유 혼합, 고유 비중↑.** 단계별 스킬 = 속성/단계 공통 풀 일부 + 개체 시그니처 다수.
- 향후 스킬 상세(효과·부여·변경)는 **별도 작업** → 그 작업이 쉽도록 `SKILL_LIB` 분리 구조(§1-2)로 받친다.

## 6. 이번 스펙의 작업 범위

**포함:**
1. `SPECIES_CATALOG` + `SKILL_LIB` 인라인 구조 신설, 기존 파생 맵 어댑터로 무회귀 연결.
2. 타입 개편(초본형 제거·버섯형 추가) + `TYPE_STATS` 갱신.
3. 마이그레이션(b): 초본형 레거시 외형/세이브 보존.
4. **시범 변환:** 버섯 개체 1~2종 + 기존 종 일부를 새 구조로 변환해 검증(전투·외형·탐사·세이브).

**제외(다음 작업):**
- 전체 개체 목록 채우기.
- 버섯 전용 메커니즘의 세부 효과 구현.
- 도트 PNG(`SPRITE_OVERRIDES`) 적용.
