# 양육/열매 시스템 설계 (#2 ⭐핵심) — 브레인스토밍 중간 박제

- **작성:** 2026-06-24
- **상태:** 🟡 브레인스토밍 진행 중 (핵심 골격 확정, 일부 열린 질문 남음 — §3 참고)
- **상위:** [master-roadmap §4-2](../../master-roadmap.md)
- **목적:** 기기 바꿔 이어갈 때 이 문서부터 읽고 브레인스토밍 재개. 미완이라 아직 구현 계획(plan)으로 넘어가지 않음.

> ⚠️ 이 설계는 기존 로드맵 결정 일부를 **수정**한다. 핵심: "생장은 경험치로만, 물/비료 폐지"였으나 → **생장과 열매를 분리**하고 물/비료는 **열매 쪽으로 부활**. (§5 결정 로그에 반영)

---

## 1. 핵심 모델 — 생장과 열매는 완전히 분리된 2축

| 축 | 동력 | 결과 | 비고 |
|---|---|---|---|
| **생장(진화)** | 경험치/레벨업 (전투·행동) | 6단계 진화(씨앗→…→완숙체) | 열매와 **무관**. 기존 시스템 |
| **열매(양육)** | **시간 경과** + **환경(물·비료·화분)** | 시간 지나면 수확 가능 | 방치형. 미네랄로 즉시 수확 |

→ 경험치/레벨업은 **생장(진화)에만** 관여, 열매와 무관. 이로써 기존 "생장=경험치" 결정은 유지하되, 열매는 별도 시간축으로 돈다.

---

## 2. 확정된 설계

### 2-1. 열매 맺힘 (생장 단계 게이트)
- **성장체(growing)부터** 열매를 맺는다. 성장체·성체·완숙체 = 열매 가능.
- 그 이전(씨앗·새싹·유체)은 **낙엽 등 부산물 = 하위 소모품**만 산출(열매 아님).
- (열린 질문) 단계가 높을수록 열매 품질/개수/주기에 점증 영향을 줄지 — §3.

### 2-2. 환경 레버 (열매에 주는 영향)
- **물/비료(투입 양)** → 열매 **질(등급) 확률**을 올린다. **상한 존재**, 보수적 — 최대치까지 줘도 **전부 S급은 안 나옴**(확률만 상승).
- **화분(품질)** → 열매 맺는 **주기 단축**(속도).
- **즉시 수확** → 유료 재화 **미네랄** 소모.
- **수확 시기·열매 개수는 식물(종/개체)별로 다름.**
- 비료 **종류**별 차등 효과, **환경**(습지/사막 등) 편향 → **보류**(확장 여지로 남김).

### 2-3. 수확 보상 — 희소도 3단계
한 열매를 까면 대부분 소모품, 가끔 스킬/변이 카드:
- **흔함: 소모품** (주 산출물) — 종류는 §3에서 협의 중
- **희귀: 변이 카드** — 그 식물의 **본인 변이형 관련**. **다른 식물에도 장착 가능**(전파/수집 경제의 공급원)
- **희귀: 본인 스킬** — 그 식물 **전용**으로 스킬 풀이 늘어남(자기강화, 전파 안 됨)
- (낙엽 = 더 하위 소모품)

→ 설계 의도: **스킬은 자기 안에 쌓고(자기강화), 변이는 밖으로 퍼뜨린다(수집·전파).**

이는 기존 시스템에 새 개념 2개를 추가한다:
1. **식물별 고유 스킬 풀 확장** (현재는 공유 `STAGE_SKILLS` 중심 → 개체가 열매로 추가 스킬 획득)
2. **변이 카드 획득처에 열매 추가** (현재는 변이형별 보급상자만 → 열매도 공급원)

## 3. 열린 질문 (재개 시 여기부터)

1. **소모품 종류 확정** — 후보: ①양육 재료(물·비료) ②생장 촉진제(경험치/레벨업 물약, `levelup_potion` 기존) ③스탯 강화제 ④재화(크레딧/미네랄 소량) ⑤화분 재료 ⑥기능성(즉시수확/생장정지).
   - **핵심 결정 대기:** 양육 재료(물·비료)가 **열매에서 나와 다시 양육에 투입되는 자급 순환 루프**를 살릴지(방치형 쾌감 ↑, 단 무한증식 방지로 보수적) vs 물·비료는 상점 크레딧으로만 구매.
2. **생장 단계의 점증 영향** — 성장체/성체/완숙체가 열매 품질·개수·주기에 차등을 줄지(키울 동기), 아니면 단순 게이트만일지.
3. **수확 단위** — 열매 1개=보상 1개 / 한 번에 여러 열매 / 묶음 개봉. (잠정: 식물별 개수 다름 = 여러 열매 롤)
4. **수치** — 기본 주기(분/시간), 미네랄 즉시수확 비용, 등급 확률 상한, 소모품:카드:스킬 비율.
5. **본인 스킬 확장 상한** — 몇 개까지? 로드아웃 6칸과의 관계(풀만 늘고 장착은 6칸 유지?).

## 4. UI / 구현 메모 (잠정)
- **양육 페이지**(하단 4번째 탭): 화분+식물(누끼)+열매 표시 + 열매까지 남은 시간 게이지 + 물/비료 투입 + 수확 버튼 + 미네랄 즉시수확.
- **종자 가방**은 양육 입구로 연결(심기 → 화분에 배치 → 양육 시작).
- **데이터:** 식물 개체에 `fruit` 상태 필드 신설(`nextFruitAt`/`careWater`/`careFert`/`fruitGrade` 등) + `normalizeState` 마이그레이션(무회귀). 수확 연출은 `#evolveModal` 패턴 재활용.
- **식물/화분 분리 렌더(#12·#3 연동):** 현재 `spriteFor`는 식물 하나만 그리고 화분은 `.pot-emoji`/배경. 식물 PNG + 화분 PNG **2레이어**(화분 입구 앵커, idle sway는 식물 레이어만)로 분리 → 양육·도트화의 공통 전제.
