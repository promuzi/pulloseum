## 1. 식물 종류 만들기

**목표:** 종(species)을 늘려 수집·육성의 다양성을 키운다.

**원칙 (일관성):**
- 각 종은 **고정 속성 1개**(불/물/풀/번개/대지/빙결/바람)를 가진다. 외형·이름·스킬이 속성과 일치해야 한다(과거 "속성·이름·외형 불일치" 버그 교훈 — 2026-06-18 로그 참고).
- 각 종은 **식물 타입**(grass/tree/flower/cactus/vine 등)으로 기본 실루엣이 갈린다.
- 종마다 **시작 스킬 셋 + 생장단계별 해금 스킬**이 속성에 맞게 부여된다.
- 변이형(포식/무기/독성/포자/발광/용족)은 종과 **독립**이며 변이 카드로 후천 부여된다. 종 자체에 변이형을 못박지 않는다.

**해야 할 일:**
- [ ] 새 종 목록 확정: 이름 / 속성 / 타입 / 콘셉트 한 줄 (표로)
- [ ] 종별 스킬 매핑이 속성 스킬 풀(`ELEMENT_SKILLS`, `ELEMENT_GROWTH_SKILLS`)과 어긋나지 않게 검토
- [ ] 외형 생성기(`composePlantSvg`/`spriteFor`)가 새 종을 자연스럽게 그리는지 확인 (도트화 시 PNG 오버라이드로 대체 — 3번 연동)
- [ ] 데이터 추가 위치: `index.html` 내 `SPECIES` / `SPECIES_SEEDTYPE` (+ 참고용 `data/alien-plant-pvp-data.js`)

**열린 질문:** 종 개수 목표(예: 속성당 2~3종)? 종별 고유 패시브를 줄지(잠재특성과 연결)?

## 2. 식물 양육 / 열매 시스템 ⭐ 핵심

**비전:** 전투와 분리된 **양육 루프**를 만든다. 식물을 키워 **열매**를 맺고, 열매에서 보상(스킬/변이 카드/크레딧 등)을 얻는다. 양육은 방치형 감성의 핵심 축이 된다.

**확정된 결정:**
- **생장은 경험치로만.** 물·비료로 단계를 올리던 방식을 폐기하고 **경험치 누적만으로 생장**하게 바꾼다. (물/비료는 제거하거나 경험치 보조재로 재정의)
- **생장 정지 아이템 추가.** 특정 생장 단계에서 **경험치 획득을 막아 그 단계에 머물게** 하는 아이템. (의도: 원하는 외형/밸런스 단계 유지, 저단계 토너먼트 파밍 등)
- **화분(pot)은 양육 전용 요소.** 전투 스탯에는 영향을 주지 않고 **양육(열매·경험치·성장 속도 등)에만** 영향을 준다.
- **페이지 재배치:** 양육을 4번째 페이지로, **탐사선은 5번째 페이지로 이동.** (기존 하단 헤더/네비 구조 재정렬 필요)

**열매 시스템 — 설계 필요:**
- [ ] **맺힘 메커니즘:** 무엇이 열매를 맺게 하나? (생장 단계 도달 / 시간 경과 / 경험치 임계 / 화분 효과 조합)
- [ ] **열매 등급:** 열매에 등급(D~S?)을 둘지, 둔다면 무엇이 등급을 결정하나(식물 잠재력/화분/단계)?
- [ ] **열매 보상:** 스킬 / 변이 카드 / 크레딧 / 미네랄 / 종자 중 무엇을, 어떤 확률·등급으로?
- [ ] **수확 루프:** 열매 수확 → 보상 → 다시 맺힘까지의 주기. 방치 보상으로 쌓이게 할지.
- [ ] **화분 종류:** 화분별 효과 차등(성장속도↑ / 열매등급↑ / 특정 보상 가중 등). 화분 획득/구매 경로.

**UI 설계 필요:**
- [ ] 양육 페이지 레이아웃 (화분 위 식물 + 열매 표시 + 수확 버튼 + 경험치 게이지)
- [ ] 화분 교체/장식 UI
- [ ] 열매 수확 연출 (5번 애니메이션 연동)

**구현 메모(현 코드 기준):**
- 현재 생장은 `applyGrowthAction`/`feedGrowth`(물/비료) + `gainGrowthExp`(전투 승리)로 이원화돼 있음 → **경험치 단일화로 통합** 필요.
- 진화 연출 `#evolveModal`은 재활용 가능.
- 식물 데이터에 `fruit` 관련 필드 신설 예정(맺힘 진행도/등급/보상 풀). `normalizeState` 마이그레이션 필수(무회귀).

## 3. 도트(픽셀아트) UI 적용

**상태:** 방향·규격은 [pixel-art-ui-roadmap.md](pixel-art-ui-roadmap.md)에 정리됨. **도트 작업 전 그 문서를 먼저 읽을 것.**

**핵심 결정 필요:**
- [ ] **식물과 화분을 분리 이미지로 만들지** — 분리하면 화분 교체(2번)와 조합이 쉽지만, 둘을 겹쳤을 때 **자연스러운 배치·움직임**이 가능한지 시범 제작으로 검증.
  - 권장: **분리**(식물 PNG + 화분 PNG 레이어). 식물은 화분 입구 기준 앵커, 미세 흔들림(idle sway)은 식물 레이어에만.
- [ ] 도트 생성 AI 도구 선정·시범 (식물 1종 → 기존 `spriteFor`의 `SPRITE_OVERRIDES`로 교체)
- [ ] 적용 범위: 식물 → 아이콘 → UI 프레임 → 애니메이션 순으로 단계적
- [ ] 애니메이션도 도트로(idle/공격/피격/진화) — 5번과 연동

**기술 훅:** `spriteFor`의 `SPRITE_OVERRIDES`가 외부 PNG 교체 장치. 코드 수정 없이 이미지만 끼우는 구조 유지.

## 4. 함선 / 길드 / 방꾸(방 꾸미기) 시스템

**상태:** 함선 내부 워킹(포켓몬식 타일 이동)은 **기초 구현됨**(2026-06-19 로그, `#shipScreen`/`SHIP_MAP`/`SHIP_FEATURES`). 콘솔 상호작용으로 기존 시스템 진입 가능.

**확장 방향:**
- [ ] **방 내부 설계 고도화:** 더 넓은 맵/여러 방, 워프 콘솔로 다른 장소 이동
- [ ] **꾸미기 → 기능 연결:** 가구를 배치하면 단순 장식이 아니라 **기능**(예: 특정 가구가 양육 속도↑, 상점 할인 등)으로 이어지게. `SHIP_FEATURES` → `state.ship.furniture`로 데이터화.
- [ ] **속도 조절:** 캐릭터 이동/연출 속도 옵션
- [ ] **캐릭터 꾸미기:** 탐사복 도트 커스터마이즈(색/파츠)
- [ ] **길드 기능:** 현재 잠금(🔒). 친구 초대/동행(드로이드 NPC가 예고 중), 길드 정거장 방문, 공동 목표 등. 8번(서버)과 연동 가능성.

**확장 포인트(이미 데이터 주도):** 맵·`SHIP_FEATURES`·NPC를 좌표/action 추가만으로 늘릴 수 있게 설계돼 있음.

**상태:** 궤도 우주맵 + 탐사선 4스탯 + 궤도 게이팅 구현됨(2026-06-18 로그). 추가 고민 필요.

**확정/고민할 결정:**
- [ ] **"행성마다 식물이 다 있는 건 이상하다"** → 행성·지역별로 **나오는 종을 제한**(테마/속성에 맞게). 이미 `cardTypes`로 카드 카테고리는 제한 중 — **종자(종)도 같은 식으로 분포 제한**.
- [ ] **탐사 거리/은하 확장:** 더 먼 은하/구역을 해금형으로. 연료탱크 레벨(궤도 게이팅)과 연결.
- [ ] **풀로세움 참가 연료·크레딧:** 토너먼트 참가에 **연료/크레딧 비용**을 둘지 결정. (탐사 연료와 통합? 별도 자원?) → 경제 루프의 핵심이므로 2번(양육 보상)·경제 밸런스와 함께 설계.
