# 식물 픽셀 아트 · 모션 제작 가이드 (그림 넣는 법)

> **이 문서의 용도:** 사용자가 직접 식물 그림(픽셀/모션)을 그려서 게임에 넣으려 할 때의 **공식 작업 방식**. 다른 기기·다른 세션에서 "모션 어떻게 넣어?" / "그림 어떻게 넣어?" 라고 물으면 **이 문서 그대로** 안내한다. (사용자가 그림을 그려서 직접 넣는 흐름이라, 매번 같은 답이 나와야 함.)
>
> 작업 파이프라인: **사용자 손그림/구상 → GPT로 그림 → 픽셀화 AI → (모션이면) 모션 프레임 AI → 게임에 삽입.**

---

## 0. 한 줄 요약

**화분은 그리지 마라.** 식물만 그리고 배경은 투명하게. 게임이 화분을 알아서 밑에 깔아준다. AI가 화분 없이는 식물을 잘 못 그리니, **임시로 "단색 분홍(#FF00FF) 화분"을 그리게 시킨 뒤 그 분홍색만 한 번에 지운다.**

---

## 1. 왜 화분을 안 그려도 되나 (게임 구조)

게임은 **식물 레이어와 화분 레이어가 완전히 분리**돼 있다. 런타임에 `spriteStack(potVisual(...), 식물그림)` 으로 **화분을 깔고 그 위에 식물만 얹는다**.

- 근거 코드: `index.html` `spriteFor()` (~10444), `composePlantSvg()` 주석 (~10245): *"식물 외형 = 식물만(내장 화분 없음·항상 분리)"*.
- 화분 종류(테라코타→도자기→유리→크리스탈→황금 5종)는 게임이 알아서 처리. **사용자는 화분을 만들 필요가 전혀 없다.**

→ 그래서 사용자가 만들 최종 결과물은 **"화분 없는 식물 + 투명 배경" 그림 하나**뿐이다.

---

## 2. 그림 1장이 만족해야 할 스펙

| 항목 | 값 |
|---|---|
| 캔버스 크기 | **120 × 140** (px) |
| 배경 | **투명** (PNG 알파) |
| 내용 | **식물만** (화분·바닥·그림자 없음) |
| 밑동 정렬 | **base y96** — 140 높이 중 위에서 96 지점이 식물 줄기 밑동(= 화분 입구). 모든 그림에서 이 위치를 고정해야 게임 화분에 정확히 접지된다. |

`object-fit:contain`으로 들어가므로 여백 비율만 맞으면 된다.

---

## 3. 게임에 넣는 법 (코드 한 줄)

`index.html`의 `SPRITE_OVERRIDES` (~10013)에 **파일 경로 한 줄**만 등록하면, 절차적 SVG 대신 사용자 그림이 뜬다. 화분은 게임이 자동으로 밑에 깐다.

```js
// 우선순위: 개체+단계 → 개체 → 타입+단계+속성  (spriteFor ~10451)
SPRITE_OVERRIDES["carno_oak.mature"] = "assets/sprites/carno_oak_mature.png"; // 특정 개체의 특정 단계
SPRITE_OVERRIDES["carno_oak"]        = "assets/sprites/carno_oak.png";        // 그 개체 전 단계 공용
SPRITE_OVERRIDES["tree_mature_fire"] = "assets/sprites/tree_mature_fire.png"; // 타입+단계+속성 (광역)
```

- 키 형식: `"종key.단계"` > `"종key"` > `"타입_단계_속성"`.
- 식물 전용, 120×140, 투명.

---

## 🛠️ 자동 도구 — 분홍 화분 제거기

**[tools/sprite-prep.html](tools/sprite-prep.html)** 를 브라우저로 열면(무설치·인터넷 불필요) 아래 §4~6을 자동으로 해준다:
- 그림(또는 모션 프레임 여러 장)을 끌어다 놓으면 → **분홍(#FF00FF)·배경 자동 제거 → 120×140·base y96 정렬 → 투명 PNG 저장**.
- **정지 개별 / 모션 공통** 정렬 모드 선택(모션 공통 = 전 프레임 같은 정렬로 흔들림 방지).
- 허용범위·밑동Y·크기·위치 슬라이더로 미세 조정, 초록 가이드선(y96)이 화분 입구.
- 모션 프레임은 **스프라이트 시트(가로 한 줄)** 로도 내보내기.

손으로 색 키잉할 필요 없이 이 도구에 넣으면 끝. (수동 절차의 원리는 아래 §4~6 참고.)

---

## 4. 핵심 트릭 — 분홍 화분 (제작 ~ 화분 제거)

AI는 식물이 허공에 떠 있으면 어색하게 그린다. 그래서:

1. **AI에게 화분을 "단색 분홍 평면 덩어리(#FF00FF)"로 그리라고 지시.** 배경도 균일한 단색.
   - 프롬프트 예: *"식물이 단색 마젠타(#FF00FF)의 단순하고 평평한 화분에 심겨 있고, 배경은 균일한 단색. 화분에는 무늬·질감·그림자 없음."*
   - AI는 "화분에 심긴 식물"이라 자연스럽게 잘 그린다.
2. 다 그린 뒤 **그 분홍색(#FF00FF)만 골라 투명으로 지운다** (색 키잉). 무늬 있는 화분과 달리 단색이라 경계가 깔끔하게 빠진다.
3. 남은 건 식물뿐 → 120×140·base y96으로 정렬 → 게임에 삽입.

---

## 5. 정지 그림 (먼저 이걸로 익힌다 — 사용자 선택)

1. GPT로 그림 (분홍 화분 + 단색 배경, §4)
2. 픽셀화 AI (캔버스 120×140, base y96 유지)
3. 분홍색 색 키잉으로 화분+배경 제거 → 투명 PNG
4. `SPRITE_OVERRIDES`에 등록 (§3)

**성장 단계 그림(씨앗→새싹→유체→성장체→성체→완숙체)은 "모션"이 아니라 단계별 그림 6장**이다. 단계마다 별도 PNG로 그리고 `"종key.단계"` 키로 각각 등록.

---

## 6. 모션 (나중에 — 제작 방식은 정지와 동일)

영상은 그림이 여러 장(프레임)이다. **분홍 화분 트릭이 모션에서 더 잘 먹힌다:** 화분 색이 모든 프레임에서 똑같은 분홍이므로, **"분홍 지워"를 한 번에 전 프레임에 일괄 적용** → 프레임마다 손으로 화분 지울 필요가 없다.

- 요령: 모션 AI에 **"화분은 가만히, 식물만 움직여"** 라고 시키면 화분이 안 움직여 더 깔끔하게 빠진다.

### ⚠️ 게임에 넣는 단계는 다름 (현재 미지원 — 추가 작업 필요)

지금 `SPRITE_OVERRIDES`는 **정지 PNG 1장 교체만 지원**한다. 움직임을 넣으려면 둘 중 하나:

- **방법 A — 스프라이트 시트:** 프레임 여러 장을 한 줄로 이어 붙인 그림 1장 + "프레임 N장" 정보. 게임이 빠르게 넘겨 재생. → 게임 코드에 **프레임 재생 기능을 새로 추가**해야 함(Claude가 해줄 수 있음).
- **방법 B — GIF/APNG:** `<img>`로 그대로 넣으면 끝(가장 간단). 단점: 색 뭉개짐·속도 조절 불가·용량 큼.

**제작(분홍 화분 그리고 지우기)은 정지든 모션이든 똑같다. 게임 삽입 마지막 단계만 다르다.** 그래서 정지로 먼저 익혀두면 모션 때 그대로 써먹는다.

---

## 7. 어떤 모션이 필요한가 (우선순위)

현재 게임은 그림 1장을 CSS로 통째 흔들어 움직임을 흉내 낸다(앞으로 쑥/움찔/살랑). 직접 그리면 훨씬 생생해진다.

**핵심 3개 (이것만 있어도 게임이 산다):**
1. **대기(Idle)** ⭐최우선 — 숨쉬듯 살랑살랑. 거의 모든 순간 보임. 2~4프레임.
2. **공격(Attack)** — 앞으로 휙/잎 휘두름. 전투 핵심.
3. **피격(Hit)** — 맞을 때 움찔. 짧게 2~3프레임.

**보너스:**
4. **승리(Win)** — 신나서 흔들흔들
5. **쓰러짐(Faint)** — 시들어 쓰러짐 (게임에 쓰러지는 효과 `collapse` 이미 있음)

**모션으로 안 만들어도 되는 것:**
- 독/화상/방어막 등 **상태 효과** → 식물 위에 따로 뜨는 별도 레이어(`spriteFx`). 식물 모션과 무관.
- **성장 단계** → 모션 아님. 단계별 그림 6장(§5).

**연습 추천:** 식물 하나로 **대기 + 공격 2개만 먼저** 만들며 "분홍 화분 → 지우기 → 정렬 → 삽입" 파이프라인을 익힐 것.
